# v1.21 Phase 80 — Sidebar Audit

Audit produced 2026-05-13. One row per visible panel section across the left sidebar, right sidebar, top bar, and floating toolbar. Decision column drives Phases 81–84 work.

## Methodology

Walked the four chrome entry points mounted in `src/App.tsx`: `TopBar` (line 191), `Sidebar` (line 201), `FloatingToolbar` (line 268, inside the canvas wrapper), `PropertiesPanel` (lines 282/314, conditionally mounted when `selectedIds.length > 0`), and `StatusBar` (line 332). Inside each, every `PanelSection`, `<section>`, and visually-bounded button group counted as one "section". The Properties panel rows are grouped by selection type (no-selection / wall / product / ceiling / custom element / stair / multi-select), and each `PanelSection` inside a selection branch is one row. The Rooms tree (`RoomsTreePanel`) counts as one section even though it expands inline — it's a single piece of IA real estate. Sections inside collapsible `PanelSection`s that ship with `defaultOpen={false}` are still considered "visible" for this audit because they show up in the scrolling stack. The Status Bar is included even though it isn't a sidebar — it competes for the same screen real estate and Jessica reads it the same way. Decisions are biased toward hiding things behind context (active tool or active selection) wherever it's safe to do so.

## Top Findings (Plain English for Jessica)

1. **The left sidebar shows every library you've ever added at once.** Custom Elements, Framed Art, and Wainscoting all stack vertically and stay visible even when you're nowhere near using them. The fix is the same rule the Window preset switcher already follows — only show the library when the tool that needs it is active. Phase 84 work.

2. **The right panel is one long scroll, not a focused editor.** When you click a wall, you see Dimensions → Position → Openings → Surface → Saved Camera → Delete all in one column. Figma solves this with tabs ("Geometry / Material / Openings"). Phase 82 is the rebuild — tabs per selection type, with the most-used controls on top.

3. **The same buttons exist in two places.** Undo and Redo appear in both the TopBar and the FloatingToolbar. The Library button is in the TopBar AND the FloatingToolbar view group. One canonical home for each — the FloatingToolbar wins because that's where your eye is when you're working in the canvas.

4. **Read-only coordinates and counts dilute the real editing surfaces.** "System Stats" in the left panel, "Position" rows showing raw x/y values, and "Material" rows that just repeat the category string all show up at the same visual weight as the editable controls. Collapse or remove them so the editable stuff stands out.

5. **The Settings button doesn't do anything yet.** TopBar line 270 has no onClick handler — clicking it is a dead end. Either wire it up in v1.21 or remove it until Phase 76 brings the real settings UI.

6. **The window preset story is buried.** When Jessica places a window, the preset chips appear floating over the canvas (good, Phase 79), but selecting an existing window dumps you into a long-scroll Properties panel where the preset is nowhere — she has to delete and re-place. The inspector rebuild in Phase 82 (IA-05) puts the preset switcher as the first control in the window inspector's "Preset" tab.

7. **"Layers" promises more than it delivers.** The left-panel "Layers" section is a single Show Grid checkbox. The grid toggle already lives in the FloatingToolbar. Remove the Layers panel entirely.

8. **The Rooms tree is doing what the left panel is supposed to do — but it's just one section among ten.** It deserves to be the spine of the left panel, with everything else collapsed below it by default and remembering its collapsed state across sessions. That's exactly what IA-02 + IA-03 specify for Phase 81.
