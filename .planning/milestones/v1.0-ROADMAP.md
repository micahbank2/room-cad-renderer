## Milestone Summary

**Decimal Phases:**

- Phase 5.1: v1.0 Integration Gaps (inserted after Phase 5 to close three integration gaps identified in milestone audit — SAVE-02 reload hydration, VIZ-05 walk respawn on room switch, VIZ-05 orbit position restore)

**Key Decisions:**

- React 18 locked (R3F v8 + drei v9 compatibility — do not upgrade to React 19)
- Fabric.js for 2D, Three.js for 3D — both read from Zustand, neither mutates the other
- Local-first, no backend — IndexedDB only via idb-keyval
- Global product library (productStore) separate from per-project cadStore — one catalog across all projects
- Optional product dimensions — image-only uploads allowed, placeholder dash rendering
- CADSnapshot v2 `{ version, rooms, activeRoomId }` with migrateSnapshot wrapping v1 into `room_main`
- active-room selector pattern (useActiveRoom hooks + activeDoc helper) — single null-guard call-site per action
- Module-level async caches (productImageCache, productTextureCache, floorTexture) — dedup concurrent loads
- History-boundary pattern for drag operations (single snapshot at mousedown + no-history per frame)

**Issues Resolved:**

- EDIT-09 2D product image rendering (was border-only)
- EDIT-07/08 product manipulation (drag-drop + rotation handles)
- EDIT-06 editable wall dimension labels
- SAVE-02 auto-save with debounce + startup hydration (via 5.1)
- SAVE-03 export PNG pipeline (wrong canvas selector fixed)
- VIZ-04 3D product textures (was placeholder boxes)
- VIZ-05 eye-level walkthrough + room-switch respawn + orbit restore (via 5.1)
- VIZ-06 scene coherence (PBR, soft shadows, procedural wood floor, Environment preset)
- LIB-03/04/05 global library + nullable dims + search
- ROOM-01/02 multi-room model + templates

**Issues Deferred:**

- Manual browser verification of Phase 1 UI interactions (VERIFICATION.md status `human_needed`)
- Nyquist compliance flag flip (all 5 phases have VALIDATION.md but flag never set to true)

**Technical Debt Incurred:**

- Floor texture cache mutates shared `.repeat` — fragile under split-view / multi-room simultaneous render
- `useAutoSave` triggers save on `activeRoomId` change (cosmetic IDB churn, not a bug)
- `export2DImage` defined in lib/export.ts but never called
- No GLTF/OBJ 3D model loading (deferred to v2 — stay with image textures per Jessica's workflow)
