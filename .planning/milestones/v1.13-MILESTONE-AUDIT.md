## Audit-fix in flight

**AUDIT-FIX-01 [resolved during audit]:** `CustomElementMesh.tsx` was missing `onContextMenu`. Phase 53 wired Wall/Product/Ceiling but never retrofitted CustomElement; Phase 54 added left-click to CustomElement but didn't add right-click. Result: right-clicking a custom element in 3D opened the Paste-only Empty Canvas menu instead of the 6-action custom element menu.

Fix: 10-line addition mirroring `ProductMesh.tsx` pattern (`onContextMenu` handler with `stopPropagation` + `preventDefault` calling `openContextMenu("custom", id, position)`). Committed as `95cbf94` before v1.13 tag.

## Notable architectural changes

- **`src/hooks/useClickDetect.ts`** — reusable click-vs-drag hook with 5px threshold. Future 3D interaction work should use this pattern.
- **`src/lib/clipboardActions.ts`** — extracted from `shortcuts.ts`. Single source of truth for copy/paste; both keyboard handler and CanvasContextMenu consume it.
- **`src/components/CanvasContextMenu.tsx`** — first canvas-context-menu primitive in the codebase; reusable pattern for future UI work.
- **`uiStore.contextMenu` + `pendingLabelFocus` slices** — store-driven menu open/close avoids prop-drilling; `pendingLabelFocus` is a clean "focus this input next render" pattern.
