## Active Requirements (all shipped)

### GLTF / 3D Model Pipeline

- [x] **GLTF-UPLOAD-01** — User can upload a `.gltf` or `.glb` file via the existing product upload flow. The file is validated (size cap, format), stored in IndexedDB with SHA-256 dedup (mirrors Phase 32 LIB-08 user-texture pattern), and the resulting `gltfId` is attached to the product. Source: [#29](https://github.com/micahbank2/room-cad-renderer/issues/29).
  - **Outcome:** Validated as designed. `src/lib/gltfStore.ts` ports the user-texture pattern cleanly. drei `useGLTF` accepts blob URLs. SHA-256 dedup confirmed working — same file uploaded across two products references one IDB entry. **Shipped Phase 55, PR [#137](https://github.com/micahbank2/room-cad-renderer/pull/137).**

- [x] **GLTF-RENDER-3D-01** — Products with `gltfId` render as the actual GLTF model in 3D, not a textured box. Loading state shows a spinner; load failure falls back to bbox box.
  - **Outcome:** Validated as designed. drei `useGLTF` is Suspense-friendly with ObjectURL paths (HIGH-confidence research). Loading + error fallbacks unified to the existing textured box (consistent UX, no new error UI to design). Selection outline via `<box3Helper>` overlay preserves PBR materials. **Shipped Phase 56, PR [#138](https://github.com/micahbank2/room-cad-renderer/pull/138).**

- [x] **GLTF-RENDER-2D-01** — Products with GLTF models render a top-down silhouette in 2D, not a textured rectangle.
  - **Outcome:** Validated. Andrew's monotone-chain convex hull written inline (~30 lines) — no new dep. `THREE.GLTFLoader.parseAsync(buf, "")` confirmed working outside React. World-transform via `scene.updateMatrixWorld(true, true)` then per-vertex `mesh.matrixWorld` projection. FIX-01-style async cache with `null` sentinel for permanent fallback. **Shipped Phase 57, PR [#139](https://github.com/micahbank2/room-cad-renderer/pull/139).**

### Integration

- [x] **GLTF-INTEGRATION-01** — Existing systems work correctly with GLTF products: Phase 31 size-override scales the model; Phase 48 saved-camera works; Phase 53 right-click menus open; Phase 54 click-to-select works. Library UI shows a small indicator that a product is GLTF-backed AND auto-renders a thumbnail when no image was uploaded.
  - **Outcome:** Validated. Phase 31/53/54 × GLTF coverage already lived in Phase 56/57 e2e — no redundant tests added. Phase 48 × GLTF was the only untested combo, closed by `e2e/gltf-integration.spec.ts` E1. Library Box badge moved to **top-LEFT** corner during research (X button collision at top-right). Auto-thumbnail mirrors Phase 45 swatch-thumbnail pattern: shared WebGLRenderer, FOV-based camera framing (`distance = (maxDim/2)/tan(fov/2) × 1.4`), 3/4 perspective view, studio lighting, in-memory cache by gltfId, full disposal traverse for all 10 PBR maps + materials. Post-merge fix for React StrictMode double-mount: `Set<() => void>` per gltfId tracks ALL pending onReady callbacks, fires each on resolve. **Shipped Phase 58, PR [#140](https://github.com/micahbank2/room-cad-renderer/pull/140).**

## Out of Scope (carried forward)

These items remained out of scope at milestone close. Some are natural v1.15 candidates; others are explicitly deferred or backlog-tracked.

| Item | Status | Reason |
|------|--------|--------|
| OBJ format support | Deferred | Older format, separate codepath; defer to v1.15+ if demand surfaces |
| GLTF animations | Deferred | Furniture rarely animated; static rendering sufficient |
| Custom material overrides on GLTF | Deferred | Use embedded PBR; override is speculative |
| Walls/ceilings/custom-elements as GLTF | Deferred | Parametric primitives; GLTF unnecessary |
| LOD / progressive loading | Deferred | Premature optimization; 25MB cap keeps load times reasonable |
| Cloud-hosted GLTF library | Deferred | Storage scope expansion; v2.0+ territory |
| Animation editing UI | Out of scope | Not applicable (no animations) |
| GLTF generation / 3D modeling tools | Out of scope | We're a viewer/placer, not a modeler |
| Phase 999.4 EXPLODE+saved-camera offset | Backlog | Carry-over from v1.11; narrow trigger |
| Phase 999.1 ceiling resize handles | Backlog | Re-deferred from v1.9 |
| Phase 999.3 per-surface tile-size override | Backlog | Re-deferred from v1.9 |
| R3F v9 / React 19 upgrade ([#56](https://github.com/micahbank2/room-cad-renderer/issues/56)) | Blocked | Gated on R3F v9 stability |
| Wall-texture flake on chromium-dev ([#141](https://github.com/micahbank2/room-cad-renderer/issues/141)) | Backlog | Pre-existing flake; surfaced during v1.14 CI but not introduced by it |
