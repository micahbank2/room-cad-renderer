## Overview

v1.0 shipped the core room-visualization MVP. v1.1 polished UX. v1.2 went beyond walls + products — adding the architectural and decorative elements that make a room feel real: ceilings, floor materials, wallpaper, wall art, wainscoting, crown molding, custom elements, and two reusable libraries (framed art, wainscoting styles). Jessica can now compose a room with the actual surfaces and accent pieces she cares about.

## Milestone Summary

**Inserted Phases:**
- Phase 17: Per-Side Wall Treatments (inserted between Phase 15 and Phase 16 after user surfaced gap during Phase 15 testing)

**Key Decisions:**
- Per-project vs global catalogs: custom elements per-project, framed art + wainscoting styles global (matches reuse pattern — Jessica's art + trim styles span all her rooms)
- Reference model for wainscoting library (vs snapshot): editing library item updates all walls using it; delete falls back to legacy
- Per-side via `{ A?, B? }` wrappers (Option A from discussion) + Y-axis rotation group in 3D: minimal schema churn, clean back-compat via migration
- Live 3D preview in wainscot builder: lazy-loaded via React.lazy to keep main bundle at 602KB

**Issues Resolved:**
- Wallpaper bleeding to all 6 wall faces (ExtrudeGeometry wrapped texture everywhere) — fixed in Phase 17 by moving wallpaper to per-face overlay planes
- Wainscoting initially shipped as textured plane — rewrote as real 3D geometry in Phase 13 after user feedback ("these should be 3D")
- CUSTOM-05 catalog persistence bug discovered in audit — patched in PR #15

**Issues Deferred (v1.3 candidates):**
- Custom element edit handles (rotate/resize/drag) — selectTool doesn't hit-test PlacedCustomElement yet
- Dims + position editors for placed framed art (stock 2'×2.5' drop)
- Per-wall knob overrides on wainscoting styles (currently library-level only)
- Copy-to-other-side buttons (wainscoting, trim)
- Per-placement frame color + material overrides
- WainscotLibrary edit-in-place UI (only create/delete currently)
