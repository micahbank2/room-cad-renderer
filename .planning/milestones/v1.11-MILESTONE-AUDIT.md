## Carry-over items (tech debt → v1.12+ or 999.x)

### AUDIT-01 [MEDIUM] — Saved camera cross-mode focus targets stale position

**Tracked as:** Phase 999.4 (filed) + GH issue (filed)

**What:** When a user saves a camera angle while in NORMAL or SOLO display mode, then switches to EXPLODE and double-clicks the same node in the rooms tree, the camera animates to the saved world-space coordinates — but EXPLODE has shifted the room +X by `cumulativeX += max(width, length) × 1.25`, so the camera focuses on empty space where the room used to be.

**Trigger conditions:** All three must be true:
1. User saves a camera in NORMAL or SOLO (room at offsetX = 0)
2. User switches to EXPLODE display mode
3. User double-clicks the saved-camera node in the tree

**Mitigation possibilities:**
- Apply `roomOffsets[roomId]` from `computeRoomOffsets()` to savedCameraPos/Target inside `focusOnSavedCamera` before dispatching
- Or: store saved cameras in room-relative coordinates, add offset at focus time

**Why deferred:** Narrow trigger conditions (cross-mode use), low real-world likelihood (Jessica's primary workflow is single-mode).
