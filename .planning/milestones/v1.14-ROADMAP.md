## Phases

### Phase 55: GLTF/GLB upload + IDB storage (GLTF-UPLOAD-01)
**Goal:** User can upload a `.gltf` or `.glb` file via the existing product upload flow; file is validated, stored in IndexedDB with SHA-256 dedup.
**Requirements:** GLTF-UPLOAD-01
**Plans:** 1/1 complete
**Built:** `src/lib/gltfStore.ts` IDB layer mirroring Phase 32 LIB-08 user-texture pattern (`saveGltf`, `getGltf`, `saveGltfWithDedup`, `findGltfBySha256`). New optional `gltfId?: string` on `Product`. AddProductModal accepts `.gltf` / `.glb` MIME types with 25MB cap. Snapshot serialization preserves `gltfId`. PR #137.

### Phase 56: Render GLTF in 3D (GLTF-RENDER-3D-01)
**Goal:** Products with `gltfId` render as the actual GLTF model in 3D, not a textured box.
**Requirements:** GLTF-RENDER-3D-01
**Plans:** 1/1 complete (4 tasks)
**Built:** `useGltfBlobUrl` hook with ref-counted ObjectURL cache (`useGLTF.clear(url)` BEFORE `revokeObjectURL`). `GltfProduct.tsx` via drei `useGLTF(blobUrl)` + `THREE.Box3().setFromObject` auto-scale (uniform smallest factor) + Y-offset so `bbox.min.y === 0`. ProductMesh refactored to branch on `gltfId` with Suspense + ErrorBoundary fallback to existing textured box. Selection outline via `<box3Helper>` overlay (preserves PBR materials). Wrapping `<group>` carries Phase 53/54 click handlers. PR #138.

### Phase 57: Top-down silhouette in 2D (GLTF-RENDER-2D-01)
**Goal:** Products with `gltfId` render as a top-down silhouette polygon in the 2D Fabric canvas.
**Requirements:** GLTF-RENDER-2D-01
**Plans:** 1/1 complete (4 tasks)
**Built:** `src/lib/gltfSilhouette.ts` — `loadGltfScene` via `THREE.GLTFLoader.parseAsync(buf, "")` (non-React load), `scene.updateMatrixWorld(true, true)` then traverse mesh vertices through `mesh.matrixWorld`, project to (x, z), Andrew's monotone-chain convex hull (~30 lines inline). FIX-01-style async cache with `null` sentinel for permanent fallback. `fabricSync.ts` branches on `product.gltfId` → `fabric.Polygon` with `rgba(202,195,215,0.15)` fill + `#7c5bf0` stroke; image-cache branch gated `!product?.gltfId`. Renamed `onImageReady` → `onAssetReady` (now serves both image + silhouette). PR #139.

### Phase 58: Library badge + auto-thumbnail + integration (GLTF-INTEGRATION-01)
**Goal:** Existing systems (Phase 31, 48, 53, 54) compose with GLTF products. Library shows Box badge + auto-thumbnail.
**Requirements:** GLTF-INTEGRATION-01
**Plans:** 1/1 complete (4 tasks)
**Built:** `src/three/gltfThumbnailGenerator.ts` — shared `THREE.WebGLRenderer` (lazy-init, mirrors Phase 45 swatch pattern). FOV-based camera framing `distance = (maxDim/2)/tan(fov/2) × 1.4` at 3/4 perspective view. Studio lighting (3 lights). Full disposal traverse for all 10 PBR maps + materials on success AND catch paths. `LibraryCard` extended with `badge?: ReactNode` slot at top-LEFT (X button stays top-right). `ProductLibrary.resolveThumbnail` enforces D-09 priority: `imageUrl` > `gltfId` thumbnail > undefined. New e2e: Phase 48 saved-camera × GLTF (the only Phase 31/48/53/54 × GLTF combo not previously verified). PR #140.

**Post-merge fix (commit `f5f6c46`):** React StrictMode double-mount discarded the first onReady closure; surviving mount never re-rendered after async compute. Fixed by tracking `Set<() => void>` per gltfId and firing all callbacks on resolve.

## Cross-phase wires (verified by audit)

```
gltfStore.getGltf(id) ──┬─→ useGltfBlobUrl       (Phase 56 — 3D render via drei)
                        ├─→ loadGltfScene        (Phase 57 — 2D silhouette compute)
                        └─→ computeGltfThumbnail (Phase 58 — library auto-thumbnail)

Product.gltfId ─────────┬─→ ProductMesh branch         → GltfProduct (3D)
                        ├─→ fabricSync renderProducts  → fabric.Polygon (2D)
                        ├─→ ProductLibrary badge slot  → lucide Box icon
                        └─→ resolveThumbnail fallback  → auto-thumbnail dataURL
```

Phase 31 `resolveEffectiveDims` drives both `GltfProduct` auto-scale AND `scaleHullToProduct` in 2D. Phase 53/54 dispatch works on the polygon via `data.placedProductId` on the `fabric.Group` wrapper (unchanged from image-only products).

## Key architectural primitives shipped

- **GLTF IDB pipeline:** `gltfStore.ts` mirrors Phase 32 user-texture pattern; SHA-256 dedup, 25MB cap, blob+sha256 schema
- **Standalone GLTF loader:** `loadGltfScene` (Phase 57) — non-React `GLTFLoader.parseAsync` reused by both silhouette and thumbnail generators
- **FIX-01 async cache pattern, hardened:** `Set<() => void>` callback tracking survives React StrictMode double-mount
- **`onAssetReady` callback:** unified re-render trigger for both image cache and silhouette cache
- **Top-LEFT badge slot on `LibraryCard`:** generic `badge?: ReactNode` prop, GLTF Box is just one consumer
