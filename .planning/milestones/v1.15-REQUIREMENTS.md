## Active Requirements (all shipped)

### Architectural Primitives

- [x] **CUTAWAY-01** — In 3D, the user can see the room interior from any camera angle. The wall closest to the camera (or any user-specified wall) becomes ghosted or invisible so it doesn't block the view. Standard CAD cutaway pattern. Source: [#21](https://github.com/micahbank2/room-cad-renderer/issues/21).
  - **Outcome:** Validated as designed. Most-opposed-normal detection algorithm + bbox-center sign convention works on rectangular rooms; severe concave shapes deferred to v1.16. Module-level scratch `THREE.Vector3` instances → zero per-frame allocation in the `useFrame` raycast loop. Constant `transparent: true` at material construction + animate `opacity` only — avoids Phase 49 BUG-02 shader-recompile pattern. Per-room cutaway state in EXPLODE mode via `Map<roomId, wallId>`. **Shipped Phase 59, PR [#142](https://github.com/micahbank2/room-cad-renderer/pull/142).**

- [x] **STAIRS-01** — User can place stairs as a new architectural primitive. Configurable rise per step, run per step, total width, and orientation. Renders as connected step boxes in 3D and as a stair-symbol polygon (with directional indicator) in 2D. Source: [#19 partial](https://github.com/micahbank2/room-cad-renderer/issues/19).
  - **Outcome:** Validated. Reconciled REQUIREMENTS text "4 cardinal directions" → continuous degrees with Shift→15° snap (matches Phase 31 product convention). `Stair` shipped as a new top-level entity (NOT a customElement.kind extension) — first new entity since Phase 33. Default 7"×11"×36"×12 (residential IBC R311). **D-04 origin asymmetry trap:** `Stair.position` is bottom-step center but `axisAlignedBBoxOfRotated` expects bbox-center; tool translates by `totalRunFt/2` along UP axis before snap, reverses on commit. **First Material Symbols D-33 allowlist expansion since Phase 33** — lucide-react has no Stairs icon; `TreeRow.tsx` joined the allowlist with documented carve-out. Snapshot v3→v4 migration. **Shipped Phase 60, PR [#143](https://github.com/micahbank2/room-cad-renderer/pull/143)** (merge-resolved with PR #144 conflicts).

- [x] **OPEN-01** — User can place wall openings beyond doors and windows: archways, pass-throughs, and niches. Extends the existing wall-opening codepath. Source: [#19 partial](https://github.com/micahbank2/room-cad-renderer/issues/19).
  - **Outcome:** Validated. `Opening.type` extended to 5 kinds + optional `depthFt?: number` for niche. Type extension only — no snapshot version bump. Archway uses `THREE.Path.absarc(centerX, shaftTopY, radius, 0, Math.PI, false)` for round semicircular top. Niche renders as separate inset mesh on interior wall face (NOT a through-hole). **Critical sign-convention trap caught by research:** niche box center offsets from front face by `+N_out × d/2` (recess INTO wall), NOT `+N_in × d/2`. Niche depth clamped to `wallThickness − 1"`. **D-11 was wrong; D-11' is correct:** Phase 53/54 don't auto-inherit for openings — they were explicitly skipped at `FabricCanvas.tsx:498`. NEW wiring added: `ContextMenuKind` extended with `"opening"`, hit-test branch + `getActionsForKind('opening')` 4-action menu + `selectable + evented` 2D overlay. Wall Cutouts toolbar dropdown mirrors WainscotPopover pattern. Round-only archway (pointed/Gothic deferred). **Shipped Phase 61, PR [#144](https://github.com/micahbank2/room-cad-renderer/pull/144).**

- [x] **MEASURE-01** — User can place measurement and annotation tools: dimension lines, free-form text labels, automatic per-room area calculation. Source: [#22](https://github.com/micahbank2/room-cad-renderer/issues/22).
  - **Outcome:** Validated. **2D-only by design** (D-03) — defer 3D dimension billboards to v1.16+. `polygonArea(walls)` shoelace winding-agnostic + connectivity check returning 0 for non-closed loops. `polygonCentroid(verts)` area-weighted formula. `measureTool.ts` mirrors `wallTool.ts:34-232` closure-state pattern — click → live preview with Phase 30 `computeSnap()` consume-only → click commits. `labelTool.ts` single-click + DOM-overlay edit mode (zIndex 30); empty-text commit removes the just-placed annotation. PropertiesPanel empty-state (lines 273-287) replaced with Room-properties branch + `AREA: XX SQ FT`. **D-11 lesson applied preemptively:** measureLine + annotation kinds added to `ContextMenuKind` from the start (no inheritance assumption). Snapshot v4→v5 chains cleanly with Phase 60. Tree integration deferred (D-12). **Shipped Phase 62, PR [#145](https://github.com/micahbank2/room-cad-renderer/pull/145).**
