### Phase 64: Wall-texture chromium-dev flake fix (BUG-04)
**Goal:** Fix the wall-user-texture-first-apply chromium-dev e2e flake (`__getWallMeshMapResolved` timeout after 2D→3D→2D→3D toggle).
**Requirements:** BUG-04
**Plans:** 1/1 complete (quick-task workflow)
**Outcome:** **Real bug found and fixed.** Two root causes:

1. **Missing useEffect cleanup in `WallMesh.tsx:218-223`** — registered material refs in the test-mode `__wallMeshMaterials` registry on mount but had no cleanup function. After 2D→3D toggle the registry pointed at a discarded material. **Same React StrictMode double-mount class as the Phase 58 thumbnail-callback bug** (`f5f6c46`). Fix: cleanup function with identity check `if (reg[wall.id] === matRefA.current) reg[wall.id] = null`.
2. **3000ms timeout too tight on chromium-dev** — with React StrictMode active, four view-mode toggles produced up to 8 effective render cycles + async texture re-resolve. Bumped to 8000ms (matches other dev-server e2e timeouts).

**Verification:** `npx playwright test e2e/wall-user-texture-first-apply.spec.ts --project=chromium-dev --repeat-each=5` → **10/10 pass** (was flaking ~50% in CI before). PR [#148](https://github.com/micahbank2/room-cad-renderer/pull/148).

**Pattern alert:** This was the **second time** the project hit the StrictMode-cleanup-missing trap (Phase 58 callback + Phase 64 registry write). Captured in CLAUDE.md as Pattern #7 with required code template (commit `4a5a883`, pre-Phase-65). Phase 65 then encountered a third instance (CeilingDimInput Enter+blur double-commit) where the same lesson applied — `editStartedRef` mirrors `skipNextBlurRef`.

### Phase 65: Ceiling resize handles (CEIL-02 — Phase 999.1 finally shipped)
**Goal:** Add edge-handle resize for ceilings. Currently users could only delete + redraw. Mirror the Phase 31 product-resize pattern, adapted for the polygon-based ceiling model.
**Requirements:** CEIL-02 (promoted from Phase 999.1, re-deferred from v1.9 twice)
**Plans:** 1/1 complete (full pipeline — discuss → research → plan → execute → verify; 6 tasks)
**Built:** `Ceiling` extended with 4 new optional fields: `widthFtOverride`, `depthFtOverride`, `anchorXFt`, `anchorYFt`. **Override-anchor model:** east/south drags use default anchors at bbox.min; west/north drags explicitly set the opposite-edge anchor (`anchorXFt = bbox.maxX` or `anchorYFt = bbox.maxY`) so the dragged edge moves with the cursor (not the opposite edge). `resolveCeilingPoints(ceiling)` resolver scales every polygon vertex from the anchor: `newP = anchor + (p - anchor) * scale`. Original `Ceiling.points` array preserved as source of truth. **L-shape ceilings supported** — proportional scaling preserves polygon shape.

`fabricSync.ts` renders 4 edge handles per selected ceiling at bbox midpoints (mirrors Phase 31 product code path). `selectTool.ts` extended with `ceilingEdgeDragInfo` module-level state, Phase 30 `computeSnap()` consume-only (with `excludeId: ceilingId`), Phase 31 single-undo transaction (`updateCeiling(id, {})` at mousedown + `*NoHistory` mid-drag). `CeilingMesh.tsx` wraps `ceiling.points` with `resolveCeilingPoints`. PropertiesPanel WIDTH/DEPTH inputs + RESET_SIZE button (`editStartedRef` pattern suppresses Enter+blur double-commit). CanvasContextMenu "Reset size" action conditional on `hasOverrides`.

**Audit gates:** `git diff origin/main src/canvas/snapEngine.ts src/canvas/buildSceneGeometry.ts` = 0 lines (Phase 30 consume-only); `git diff origin/main src/types/product.ts` = 0 lines (Phase 31 untouched); snapshot version literal still 5 (additive optional fields, no bump per Phase 61 OPEN-01 precedent). Formal VERIFICATION 13/13 PASS. PR [#149](https://github.com/micahbank2/room-cad-renderer/pull/149).

## Lessons learned (process)

1. **`/gsd:quick` is the right tool for known-scope debt fixes.** 3 of 4 phases used the quick workflow successfully. Total ~55-minute wall-clock for the milestone vs. ~6-10 days for similar-sized feature milestones. The pattern: SUMMARY.md instead of CONTEXT+RESEARCH+PLAN+VERIFICATION when the diagnosis is concrete and design choices are minimal.

2. **Investigate before fixing.** Phase 63 confirmed the cascade didn't reproduce locally before changing code. Phase 66 confirmed Phase 42 had already done most of the work before scoping new types/actions. Both saved hours of unnecessary work.

3. **Capture recurring patterns in CLAUDE.md.** The StrictMode useEffect-cleanup trap was hit twice (Phase 58 + Phase 64) before being captured as Pattern #7. Phase 65 then hit a third instance (CeilingDimInput Enter+blur double-commit) and applied the lesson preemptively. Documenting the trap turned a recurring bug-class into a known pattern with a code template.

4. **Backlog re-deferral is a signal.** Phase 999.1 + 999.3 had been re-parked from v1.9 → backlog → v1.10 → backlog → … each time we picked features over debt. v1.16 was the first explicit "stop adding features, clear the parked items" milestone. Both shipped without scope creep.
