# Requirements Archive — Room CAD Renderer v1.0

**Archived:** 2026-04-05
**Milestone:** v1.0 Room Visualization MVP
**Status:** SHIPPED — all 14 active requirements validated end-to-end

## Core Value

Jessica can see her future room with her actual furniture before spending money.

---

## v1 Requirements — All Complete

### 2D Editor

- [x] **EDIT-01**: User can create a room with real dimensions (width, length, wall height) — validated (existing)
- [x] **EDIT-02**: User can draw walls with snap-to-grid (0.5ft default) — validated (existing)
- [x] **EDIT-03**: User can place doors and windows on walls — validated (existing)
- [x] **EDIT-04**: User can select, move, and delete walls and products — validated (existing)
- [x] **EDIT-05**: User can undo/redo all actions (50-level history) — validated (existing)
- [x] **EDIT-06**: User can edit wall dimensions by double-clicking the dimension label — validated (Phase 1, Plan 04)
- [x] **EDIT-07**: User can drag products from library onto the canvas — validated (Phase 1, Plan 02)
- [x] **EDIT-08**: User can rotate placed products via drag handles in 2D — validated (Phase 1, Plan 03)
- [x] **EDIT-09**: Product images render visibly on the 2D canvas — validated (Phase 1, Plan 01)

### Product Library

- [x] **LIB-01**: User can upload product image with name, category, and dimensions — validated (existing)
- [x] **LIB-02**: User can browse and filter products by category — validated (existing)
- [x] **LIB-03**: Product library is global — persists across all room projects — validated (Phase 2)
- [x] **LIB-04**: Product dimensions are optional (image-only upload allowed) — validated (Phase 2)
- [x] **LIB-05**: User can search products by name — validated (Phase 2)

### 3D Visualization

- [x] **VIZ-01**: User can see walls extruded in 3D with lighting and shadows — validated (existing)
- [x] **VIZ-02**: User can orbit the 3D camera — validated (existing)
- [x] **VIZ-03**: User can toggle between 2D, 3D, and split views — validated (existing)
- [x] **VIZ-04**: Products render in 3D with their uploaded image as texture — validated (Phase 3, Plan 01)
- [x] **VIZ-05**: Eye-level camera walkthrough to feel the room from inside — validated (Phase 4 + Phase 5.1 fixes)
- [x] **VIZ-06**: Smooth 3D experience (PBR materials, soft shadows, floor texture) — validated (Phase 3, Plan 02)

### Persistence

- [x] **SAVE-01**: User can save and load projects from browser storage — validated (existing)
- [x] **SAVE-02**: Auto-save with debounce — validated (Phase 1, Plan 05 + Phase 5.1 startup hydration)
- [x] **SAVE-03**: User can export 3D view as PNG image — validated (Phase 3, Plan 03)

### Rooms

- [x] **ROOM-01**: User can create multiple rooms within one project — validated (Phase 5)
- [x] **ROOM-02**: Room templates (living room, bedroom, kitchen presets) — validated (Phase 5, Plan 01)

## v2 Requirements (Deferred)

- iPad / touch-friendly support
- Connected whole-house floor plan (rooms linking via doorways)
- Backend / cloud sync for cross-device access
- GLTF/OBJ 3D model upload for products

## Out of Scope

- Multi-user / collaboration
- Export to CAD file formats
- Pricing / shopping list integration
- Professional drafting features (layers, annotations, blueprints)
- Mobile phone support

## Notes

- **5.1 integration gaps**: Three gaps from the v1.0 milestone audit were closed in Phase 5.1:
  - SAVE-02: App.tsx now hydrates last-saved project on mount (was stranded in IndexedDB)
  - VIZ-05: WalkCameraController re-spawns on activeRoomId change (was stuck at prior coords)
  - VIZ-05: Orbit camera position restored on walk→orbit toggle (was target-only)
- **Nyquist validation**: All 5 primary phases have VALIDATION.md files but the `nyquist_compliant` flag was never flipped to true — formal sign-off deferred.
- **No requirements were dropped or invalidated** — every active requirement shipped.
