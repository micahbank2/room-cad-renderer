# Requirements — v1.20 Surface Depth & Architectural Expansion

Deepens realism and precision: PBR maps make materials tactile, window presets give openings real-world sizing, parametric controls enable exact product placement, and columns add structural elements Jessica actually has to plan around.

Continues phase numbering from 77 → starts at **Phase 78**. Four-phase milestone.

Source issues: [#81](https://github.com/micahbank2/room-cad-renderer/issues/81) (PBR maps), [#20](https://github.com/micahbank2/room-cad-renderer/issues/20) (window presets), [#28](https://github.com/micahbank2/room-cad-renderer/issues/28) (parametric controls), [#19](https://github.com/micahbank2/room-cad-renderer/issues/19) (columns).

## Active Requirements

### PBR Maps

- [ ] **PBR-01** — User can upload an AO (ambient occlusion) map on a material. Extends the existing Upload Material modal which already handles color, roughness, and reflection map slots. Adds `aoMapId` field to the `Material` type following the same pattern as `roughnessMapId`.
  - **Verifiable:** Open Upload Material modal → see an "AO Map" upload slot → upload a grayscale JPEG → save → re-open material → AO map still attached.
  - **Phase:** Phase 78.

- [ ] **PBR-02** — User can upload a displacement map on a material. Same extension as PBR-01 — adds `displacementMapId` to `Material` type and a "Displacement Map" slot in the upload modal.
  - **Verifiable:** Open Upload Material modal → see a "Displacement Map" upload slot → upload → save → re-open → displacement map still attached.
  - **Phase:** Phase 78.

- [ ] **PBR-03** — The 3D rendering applies AO and displacement maps when a material has them. In Three.js, the mesh material receives `aoMap` and `displacementMap` props alongside the existing `roughnessMap` — WallMesh, FloorMesh, CeilingMesh, and ProductMesh are all in scope.
  - **Verifiable:** Apply a material with AO + displacement maps to a wall in 3D → surface shows visible occlusion in corners and micro-texture depth compared to the same material without those maps.
  - **Phase:** Phase 78.

- [ ] **PBR-04** — The material card in the library shows map-presence indicators when AO and/or displacement maps are present on a material, so Jessica can tell at a glance which materials are fully-specified PBR vs. basic color-only.
  - **Verifiable:** Upload a material with AO and displacement maps → its library card shows small "AO" and "DISP" badges (or equivalent visual indicators). A color-only material shows no such badges.
  - **Phase:** Phase 78.

### Window Presets

- [ ] **WIN-01** — A preset size picker (2×3 ft, 3×4 ft, 4×5 ft, Custom) appears when placing a window via the Window tool. The user picks a preset before the window is committed to the canvas. Builds on the existing `Opening` type — no data model changes needed.
  - **Verifiable:** Activate Window tool → click on a wall → a preset picker appears → choose "3×4 ft" → window is placed with width 3 ft, height 4 ft as confirmed by its PropertiesPanel values.
  - **Phase:** Phase 79.

- [ ] **WIN-02** — The window preset is visible and editable in PropertiesPanel after placement. Selecting an existing window shows the preset picker in PropertiesPanel — choosing a new preset updates dimensions with a single undo entry; choosing Custom enables free numeric input.
  - **Verifiable:** Select a placed window → PropertiesPanel shows a preset selector → change from "3×4" to "4×5" → window updates in both 2D and 3D → Ctrl+Z reverts to 3×4.
  - **Phase:** Phase 79.

### Parametric Product Controls

- [ ] **PARAM-01** — User can type exact width and depth (in feet) for a placed product in PropertiesPanel. The inputs write to the existing `widthFtOverride` and `depthFtOverride` fields on `PlacedProduct` (shipped in Phase 31). No new schema changes needed.
  - **Verifiable:** Select a placed sofa → PropertiesPanel shows "Width (ft)" and "Depth (ft)" inputs pre-filled with current values → type "8.5" in Width → sofa resizes to exactly 8.5 ft wide in 2D and 3D.
  - **Phase:** Phase 80.

- [ ] **PARAM-02** — User can type exact X and Y position (in feet from room origin) for a placed product in PropertiesPanel. The inputs write to the product's position in the store.
  - **Verifiable:** Select a placed chair → PropertiesPanel shows "X position" and "Y position" inputs pre-filled with current position → type "3" in X → chair moves to exactly 3 ft from room origin on the X axis.
  - **Phase:** Phase 80.

- [ ] **PARAM-03** — Each parametric edit (width, depth, X, Y) is a single undo entry. Ctrl+Z reverts to the value before the input was focused.
  - **Verifiable:** Type a new width value → press Enter → undo history grows by exactly 1 → Ctrl+Z restores the previous width.
  - **Phase:** Phase 80.

### Columns & Pillars

- [ ] **COL-01** — User can place a round or rectangular column with configurable size. A column placement flow (new tool or dialog triggered from toolbar) lets Jessica choose shape (round vs. rectangular), set diameter or width/depth, and click to place it on the canvas.
  - **Verifiable:** Activate the column placement flow → choose "Round", diameter "1 ft" → click on canvas → a column appears at that position.
  - **Phase:** Phase 81.

- [ ] **COL-02** — A placed column renders correctly in both 2D (footprint shape) and 3D (extruded pillar). In 2D, the column appears as a filled circle or rectangle at the correct position and size. In 3D, the column extrudes as a pillar at room wall height.
  - **Verifiable:** Place a round column → 2D view shows a circle at the correct size and position → switch to 3D → a vertical cylindrical pillar appears at full wall height.
  - **Phase:** Phase 81.

- [ ] **COL-03** — A placed column is selectable, movable, and deletable. PropertiesPanel shows shape, size, and position fields. All interactions follow the existing single-undo contract.
  - **Verifiable:** Click a column in 2D → it shows a selection indicator → PropertiesPanel shows shape/size/position fields → drag the column to a new position → Ctrl+Z moves it back → press Delete → column is removed.
  - **Phase:** Phase 81.
