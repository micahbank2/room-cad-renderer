    IMPORTANT: paintStore does NOT use idb-keyval. It does NOT have its own persistence. Custom paints live in cadStore.customPaints (part of CADSnapshot) so they participate in undo/redo and project save/load automatically. paintStore is a thin convenience layer that subscribes to cadStore and exposes customColors for React components.

    ```typescript
    import { create } from "zustand";
    import type { PaintColor } from "@/types/paint";
    import { useCADStore } from "./cadStore";

    interface PaintState {
      customColors: PaintColor[];
    }

    export const usePaintStore = create<PaintState>()(() => ({
      customColors: [],
    }));

    // Sync from cadStore — paintStore is a derived view
    useCADStore.subscribe((state) => {
      const customs: PaintColor[] = (state as any).customPaints ?? [];
      if (customs !== usePaintStore.getState().customColors) {
        usePaintStore.setState({ customColors: customs });
      }
    });
    ```

    Consumers that need to ADD or REMOVE custom paints call cadStore actions directly:
    - `useCADStore.getState().addCustomPaint(item)` — returns id, pushes history
    - `useCADStore.getState().removeCustomPaint(id)` — pushes history

    This eliminates the dual-storage problem: there is ONE source of truth (cadStore.customPaints) and undo rolls back both paint data and custom colors atomically.

    ```typescript
    import { FB_COLORS } from "@/data/farrowAndBall";
    import type { PaintColor } from "@/types/paint";

    export function resolvePaintHex(
      paintId: string,
      customColors: PaintColor[],
      fallback = "#f8f5ef"
    ): string {
      const fb = FB_COLORS.find((c) => c.id === paintId);
      if (fb) return fb.hex;
      const custom = customColors.find((c) => c.id === paintId);
      if (custom) return custom.hex;
      return fallback;
    }
    ```
