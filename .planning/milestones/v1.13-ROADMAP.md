## Phases

### Phase 53: Right-click context menus on canvas objects (CTXMENU-01)
**Goal:** Right-click on any canvas object opens a kind-specific context menu.
**Requirements:** CTXMENU-01
**Plans:** 1/1 complete (5 tasks)
**Built:** Single `CanvasContextMenu` component with `getActionsForKind()` registry. Per-kind action sets (Wall: 5, Product: 6, Ceiling: 3, Custom: 6, Empty: 0/1). 2D Fabric `mousedown` button=2 + `getObjects()` scan (avoids Phase 25 PERF-01 `evented:false` pitfall). 3D R3F per-mesh `onContextMenu` + Canvas-level empty-canvas handler. Auto-flip near viewport edges. 5 close paths (Escape / backdrop / item-click / right-click-elsewhere / resize-or-scroll). Inert when typing in form input.
**Architectural primitive:** `src/lib/clipboardActions.ts` â€” extracted from shortcuts.ts; both keyboard handler AND CanvasContextMenu consume it.

### Phase 54: PropertiesPanel in 3D + click-to-select (PROPS3D-01)
**Goal:** Click a wall / product / ceiling / custom element in 3D to select it; PropertiesPanel updates.
**Requirements:** PROPS3D-01
**Plans:** 1/1 complete (4 tasks)
**Scope reality:** Issue title was "Properties panel in 3D" but the actual gap was bigger â€” no 3D click-to-select wiring existed at all. PropertiesPanel itself was already viewMode-agnostic; once selection works, the panel works.
**Built:** `src/hooks/useClickDetect.ts` â€” shared hook with 5px drag-threshold guard distinguishes click from orbit-drag. Applied to all 4 mesh components. Canvas-level `onPointerMissed` clears selection with same drag guard. Phase 53 right-click coexists (different DOM events: `pointerup` vs `contextmenu`; different button checks).
