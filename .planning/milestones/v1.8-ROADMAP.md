## Key Accomplishments

1. **User-uploaded textures (LIB-06/07/08)** — Jessica drops a JPEG/PNG/WebP, names it, sets real-world tile size in feet+inches (reusing Phase 29 parser), applies it to walls/floors/ceilings from the material picker. 2048px longest-edge client-side downscale + SHA-256 dedup to single IDB entry. Snapshots reference textures by `userTextureId` only — zero `data:` substrings >10KB and zero Blobs in JSON. Orphan-deletion fallback to base hex color (no crash). 107/107 unit tests pass.
2. **Camera presets (CAM-01/02/03)** — Eye-level (5.5 ft) / top-down (Y = 1.5× max(roomW, roomL)) / 3-quarter (literal v1.7.5 baseline) / corner (opposite-corner ceiling height − 0.5 ft) accessible via 4 lucide Toolbar buttons + bare `1`/`2`/`3`/`4` hotkeys. ~600ms easeInOutCubic tween with imperative `OrbitControls.enableDamping` toggle, live-capture cancel-and-restart on mid-tween interrupts, view-mode + walk-mode cleanup, reduced-motion instant-snap. Full guard chain: activeElement / viewMode / cameraMode / modifiers. Zero `cadStore.past` pollution, zero `useAutoSave` triggers. 12/12 e2e specs pass on chromium-dev + chromium-preview.
3. **VIZ-10 permanent regression guard** — Playwright instrumentation harness across 4 surfaces (wallpaper / wallArt / floor / ceiling) × 2 projects (chromium-dev + chromium-preview production-minified bundle), 5-cycle 2D↔3D toggle test with within-run pixel-diff (cycle-N vs cycle-1, ≤1% delta via `pixelmatch`). [ROOT-CAUSE.md](../phases/36-viz-10-regression/ROOT-CAUSE.md) documents the no-repro outcome (Branch B per R-04): same texture UUID across 5 mount cycles, 14 goldens byte-identical. All 4 Phase 32 defensive-code pieces classified KEEP. CI workflow runs on every PR via `.github/workflows/e2e.yml`. Issue #94 stays OPEN by design — no-repro ≠ fix.
4. **Tech-debt sweep (DEBT-01/02/03/04)** — All 4 v1.6 carry-over GH issues (#44/#46/#50/#60) verified CLOSED with proper closing comments. Orphan `SaveIndicator.tsx` deleted (zero importers). `effectiveDimensions` → `resolveEffectiveDims` migration finalized with `@deprecated` JSDoc and 3 unused imports removed. Phase 29 `29-03-SUMMARY.md` frontmatter backfilled with `requirements: [EDIT-20, EDIT-21]`. 6 pre-existing vitest failures (LIB-03/04/05 + App.restore × 3) formally accepted as permanent in `deferred-items.md`.
5. **Within-run pixel-diff testing pattern** — Phase 36 surfaced and solved Playwright's platform-coupled snapshot problem. Stored `toHaveScreenshot` goldens are OS-suffixed (`-darwin.png` vs `-linux.png`), breaking CI when goldens are generated locally. New pattern: capture cycle-1 as in-memory `Buffer` baseline, compare cycles 2..N to it via `pixelmatch` + `pngjs`, assert `mismatchRatio ≤ threshold`. Helper at `tests/e2e/playwright-helpers/pixelDiff.ts`. Reusable for any "stability across actions" test.
6. **Root-cause-before-fix discipline** — Phase 36 enforced structurally via `depends_on: [36-01]` gating Plan 36-02 on `ROOT-CAUSE.md` evidence. Saved a 4th speculative fix attempt after three earlier Phase 32 attempts (Plans 05/06/07) failed to identify the root cause. Harness now catches future regressions automatically.
