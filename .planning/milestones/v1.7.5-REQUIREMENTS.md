# v1.7.5 Requirements — Design System & UI Polish

**Milestone goal:** Ship 8 Pascal Editor competitive-audit items as a cohesive design-system upgrade covering typography, spacing/iconography, collapsible sections, floating selection toolbar, gesture hints, rotation presets, inline-editable titles, and unified library card pattern.

**Requirements source:** Tracked directly on [GitHub milestone v1.7.5](https://github.com/micahbank2/room-cad-renderer/milestone/8) (issues #83–#90) rather than in `REQUIREMENTS.md`. This was intentional per the "Issues-as-Living-System" rule established 2026-04-21 — for a single-phase, well-scoped polish milestone, GH issues were the source of truth and REQUIREMENTS.md was not re-bootstrapped for this cycle.

**Success measure:** Jessica perceives the app as peer-grade to Pascal Editor on typography, spacing/iconography, properties-panel ergonomics, selection affordances, and library consistency. No regressions to existing features.

---

## Requirements

### Typography (GH #83)

- [x] **GH [#83](https://github.com/micahbank2/room-cad-renderer/issues/83)** — Mixed-case hierarchy replaces blanket UPPERCASE in section/panel/button labels; monospace (IBM Plex Mono) reserved for dynamic identifiers, unit values, and status strings; hierarchy has ≥3 visually distinct levels.
  - **Satisfied by:** Plans 33-01 (tokens), 33-02 (typography pass)
  - **Verifiable:** Every header/button/label in Toolbar/Sidebar/PropertiesPanel reads in mixed case; UPPERCASE retained for `WALL_SEGMENT_{id}`, `6 ft 3 in`, `SAVED` / `SAVING`.

### Properties Panel Ergonomics (GH #84)

- [x] **GH [#84](https://github.com/micahbank2/room-cad-renderer/issues/84)** — Properties panel sections collapse/expand via chevron affordance; per-section open state persists across session via localStorage.
  - **Satisfied by:** Plan 33-04 (`CollapsibleSection` primitive + 10 sections wrapped)
  - **Verifiable:** Collapse Dimensions → reload page → section stays collapsed.

### Selection Affordances (GH #85)

- [x] **GH [#85](https://github.com/micahbank2/room-cad-renderer/issues/85)** — Selecting a product, wall, or custom-element surfaces a floating mini-toolbar with Move/Duplicate/Delete; positioned relative to selection bbox without occluding edit handles; hides during active drag.
  - **Satisfied by:** Plan 33-06 (`FloatingSelectionToolbar` + `uiStore.isDragging` bridge)
  - **Verifiable:** Click a placed product → toolbar appears above bbox → drag starts → toolbar hides → drop → toolbar reappears.

### Canvas Hints (GH #86)

- [x] **GH [#86](https://github.com/micahbank2/room-cad-renderer/issues/86)** — Canvas shows a gesture affordance chip ("Pan • Rotate • Zoom" with modifier hints) in a non-intrusive corner; dismissible; hides during active drag; dismissal persists across reload.
  - **Satisfied by:** Plan 33-07 (`GestureChip` 2D/3D variants + localStorage dismiss)
  - **Verifiable:** Chip visible on first mount → dismiss → reload → chip stays dismissed.

### Rotation Presets (GH #87)

- [x] **GH [#87](https://github.com/micahbank2/room-cad-renderer/issues/87)** — PropertiesPanel rotation row includes `-90°` / `-45°` / `0°` / `+45°` / `+90°` preset chips beside the numeric input; clicking each pushes exactly one undo entry.
  - **Satisfied by:** Plan 33-08 (preset chips for products + custom elements)
  - **Verifiable:** Click `+45°` → rotation applies → single `Ctrl+Z` fully reverts (store-level behavior test confirms `past.length++1`).

### Inline-Editable Titles (GH #88)

- [x] **GH [#88](https://github.com/micahbank2/room-cad-renderer/issues/88)** — Document title (Toolbar) and room tab labels are click-to-edit inline with the same UX as Phase 31 label override (Enter/blur commits, Escape cancels). ProjectManager demoted to read-only display.
  - **Satisfied by:** Plan 33-09 (`InlineEditableText` primitive + Toolbar doc title relocation + RoomTabs inline-edit)
  - **Verifiable:** Click doc title → type new name → Enter → name commits + persists.

### Unified Library Card (GH #89) — PARTIAL BY DESIGN

- [x] **GH [#89](https://github.com/micahbank2/room-cad-renderer/issues/89)** — Product Library / Art Library / Wainscot Library / Material picker share a unified `LibraryCard` + `CategoryTabs` component; visually identical structure.
  - **Satisfied by:** Plan 33-05 (`LibraryCard` + `CategoryTabs` primitives)
  - **Coverage:** 2 of 5 surfaces migrated (ProductLibrary, CustomElementsPanel). WainscotLibrary, Paint/Material picker, FramedArt library NOT migrated — deferred per **D-31** (shape mismatch between existing data models and LibraryCard shape). Tracked as follow-up PRs under GH #89 (issue kept open).
  - **Verifiable:** ProductLibrary + CustomElementsPanel render through `<LibraryCard>`; visual regression baseline matches.

### Spacing + Iconography (GH #90)

- [x] **GH [#90](https://github.com/micahbank2/room-cad-renderer/issues/90)** — Spacing scale + border-radius + icon size/stroke are audited and normalized across Toolbar / Sidebar / PropertiesPanel / modals; zero ad-hoc arbitrary values remain in the 4 highest-traffic files.
  - **Satisfied by:** Plans 33-01 (tokens + radius-lg sweep), 33-03 (4-file zero-arbitrary audit + `lucide-react` chrome icons + Material Symbols CAD-glyph allowlist)
