# Requirements — v1.21 Sidebar IA & Contextual Surfaces

Rebuild the side panels and floating toolbar from the information-architecture up. The current sidebars are a flat list of every section the app has ever added — Rooms, Materials, Custom Elements, Art Library, Wainscot Library, System Stats, Layers, Snap, Properties — all visible at once, with no priority and no awareness of what the user is currently doing.

Target: an enterprise-grade canvas-app sidebar comparable to Figma (Design / Prototype / Inspect tabs), Miro (contextual right rail per object type), or Linear (collapsible groups with sensible defaults). The user should only see what's relevant to the **active tool** and the **current selection**, with progressive disclosure for everything else.

**Out of scope:** new features. This is purely a layout, IA, and contextual-visibility overhaul. No new tools, no new product capabilities.

**Source feedback:** Jessica UAT session 2026-05-13 (Phase 79 verification) — "the UI UX layout, structure etc sucks for the left and right sidebars — it's hard to read, it's a fuck ton of stuff smashed together, it doesn't have any smart logic behind it where things show up or go away based on things done."

## Active Requirements

### Discovery + Audit

- [ ] **IA-01** — Produce a sidebar audit: every section currently visible in the left and right panels, what it does, which user task it serves, and how often Jessica needs it (always / per-tool / per-selection / rarely). The output is a single Markdown table that becomes the input to the IA decisions in IA-02.
  - **Verifiable:** `.planning/milestones/v1.21-SIDEBAR-AUDIT.md` exists with one row per visible sidebar section, columns: name, location (left / right), purpose, tied-to-tool, tied-to-selection, frequency, decision (keep visible / collapse / move / remove).
  - **Phase:** Phase 80.

### Left Panel (Project Structure)

- [ ] **IA-02** — Group left-panel sections into a clear hierarchy with persistent state. The Rooms tree (with Walls, Ceiling, Products, Custom Elements, Stairs underneath the active room) sits at the top. Below it, secondary panels collapse by default and remember their collapsed state across sessions: Room Config, Materials, System Stats, Layers, Snap, Custom Elements catalog, Art Library, Wainscot Library.
  - **Verifiable:** Open the app fresh → left panel shows the Rooms tree expanded and all secondary panels collapsed. Expand Materials → reload page → Materials panel is still expanded.
  - **Phase:** Phase 81.

- [ ] **IA-03** — The Rooms tree gets the Figma layers-panel treatment: visibility toggle per node, hover highlights the corresponding object on canvas, click selects it, double-click renames inline. Walls, products, custom elements, and stairs are leaf nodes — not separate panel sections.
  - **Verifiable:** Hover "North wall" in the tree → north wall briefly glows on the 2D canvas. Click it → wall is selected, properties panel mounts on the right. Click the eye icon → wall hides from canvas but stays in tree.
  - **Phase:** Phase 81.

### Right Panel (Inspector)

- [ ] **IA-04** — The right panel becomes a **contextual inspector** that only mounts when something is selected. Empty selection → panel collapses to a thin rail with no content. Selection → panel slides open with the relevant inspector tabs for the selected object type (Wall vs. Window vs. Product vs. Ceiling vs. Custom Element).
  - **Verifiable:** Click empty canvas → right panel rail shows but no inspector body. Click a wall → inspector body appears with `Geometry / Material / Openings` tabs. Click a window → inspector body appears with `Preset / Dimensions / Position` tabs.
  - **Phase:** Phase 82.

- [ ] **IA-05** — The Phase 79 Window Preset switcher lives in the inspector's `Preset` tab when a window is selected, not buried in a scrolling Properties section. The active preset chip + Custom toggle is the top control in that tab.
  - **Verifiable:** Place a window → click it → right inspector opens to a `Preset` tab → 5 preset chips + Custom are the first visible control. Switching the preset still produces a single Ctrl+Z revert (Phase 79 invariant preserved).
  - **Phase:** Phase 82.

### Floating Toolbar

- [ ] **IA-06** — The floating toolbar gets bigger hit targets (44 px minimum, currently ~28 px), grouped sections separated by clearer dividers, and tool labels visible on hover instead of just icons. Responsive collapse: on viewports under 1280 px wide, the toolbar wraps into two rows instead of overflowing or hiding tools.
  - **Verifiable:** Open the app on a 1024×768 viewport → all tools visible across two stacked rows of the floating toolbar, no horizontal scroll, every tool clickable. Hover any tool → its name appears as a label below the icon.
  - **Phase:** Phase 83.

- [ ] **IA-07** — Tool groups are visually banded: drawing tools (select, wall, door, window, opening), measurement (ruler, dimension, label), structural (column, stair, opening fixture), view modes (2D / 3D / Split / Library), and utility (grid, snap, zoom, fit-to-screen). Each group has a thin label above it in mixed case ("Drawing", "Measure", "Structure", "View", "Utility").
  - **Verifiable:** Open the toolbar → see 5 visually distinct groups with labels above each group. Library is back in the View group (regression fix landed in v1.20 PR — invariant for v1.21).
  - **Phase:** Phase 83.

### Contextual Visibility Rules

- [ ] **IA-08** — Tool-specific surfaces only mount when the relevant tool is active. The Window Preset switcher (Phase 79) is the precedent: it mounts only when `activeTool === "window"`. Apply the same rule to Wainscot Library (only when wainscot tool active), Custom Elements catalog (only when custom-element tool active), and any future tool-specific chrome. Selecting an object on canvas should NOT auto-switch tools.
  - **Verifiable:** Activate Wall tool → Wainscot Library section is hidden in the left panel. Activate Wainscot tool → Wainscot Library appears. Selecting a wainscot panel on canvas while in Wall tool does NOT switch tools, but DOES surface the wainscot inspector in the right panel.
  - **Phase:** Phase 84.

## Success Criteria (Milestone-Level)

A new user (Jessica's mom, say) opens the app and within 30 seconds can:
1. Tell which sidebar section is "what's in my room" vs "what I'm currently editing".
2. Find the preset switcher when placing a window without being told where it is.
3. Find Library mode without scanning the entire screen.
4. Tell when a panel is hidden because it's not relevant vs. hidden because it's collapsed.
