## Typography

5-tier ramp inherited from Phase 33; tree only uses 2 of the 3 size tokens.

| Role | Size | Weight | Line Height | Family | Where in tree |
|------|------|--------|-------------|--------|---------------|
| Section header | 16px (`text-base`) | 500 (`font-medium`) | default | IBM Plex Mono (`font-mono`) | "Rooms" header label (matches existing CollapsibleSection in Sidebar.tsx:32) |
| Room name (depth 0) | 13px (`text-base` token = `--text-base`) | 500 (`font-medium`) | default (1.5 inherited) | IBM Plex Mono | Top-level room rows |
| Group header (depth 1) | 11px (`text-sm` token = `--text-sm`) | 500 (`font-medium`) | default | IBM Plex Mono | "Walls", "Ceiling", "Products", "Custom elements" group labels |
| Leaf label (depth 2) | 11px (`text-sm` token = `--text-sm`) | 400 (default) | default | IBM Plex Mono | Individual wall / product / ceiling / custom-element rows |
| Empty-state copy | 11px (`text-sm`) | 400 | default | IBM Plex Mono | "No walls yet" etc. â€” italic via `italic` utility |

**Casing rules (D-03 inherited):**
- **Mixed case** for static UI labels: `Rooms`, `Walls`, `Ceiling`, `Products`, `Custom elements`, room names as user typed them, group headers.
- **Mixed case** for derived semantic wall labels: `North wall`, `East wall`, `South wall`, `West wall`. Diagonal fallback: `Wall 3` (mixed case + numeric index, NOT uppercased). This is a deliberate departure from the legacy `WALL_SEGMENT_{id}` UPPERCASE pattern â€” these are human-readable derived labels, not raw CAD identifiers.
- **UPPERCASE** is NOT used in the tree. No dynamic CAD identifier tokens surface here (the tree shows derived labels, not raw IDs).

## Interaction Contract

### Selection + Focus (single click on a row's label area)

| Node type | Selection behavior | Camera behavior |
|-----------|--------------------|-----------------|
| Room (depth 0) | `useUIStore.select([roomId])` | `cadStore.switchRoom(roomId)` AND dispatch Phase 35 `pendingPresetRequest` for room-bbox framing (D-10) |
| Group header (depth 1: Walls/Ceiling/Products/Custom elements) | NO selection change | NO camera change â€” group headers ONLY toggle expand/collapse on click |
| Wall leaf | `useUIStore.select([wallId])` | Reuse MIC-35 wall-side framing path (D-07) |
| Product / custom-element leaf | `useUIStore.select([placementId])` | Compute bbox, dispatch `pendingPresetRequest` at 1.5Ă— diagonal (D-08) |
| Ceiling leaf | `useUIStore.select([ceilingId or synthetic key])` | Tilt-up tween from floor center (D-09) |

**Rationale for group-header click being expand-only:** Selection has no meaning at the "all walls" level â€” `selectedIds` is a flat array of leaf ids. Tying group click to expand-only also matches the user's expand/collapse mental model.

### Visibility toggle (single click on the eye-icon button)

- `uiStore.toggleHidden(id)` â€” adds/removes from `hiddenIds: Set<string>`.
- Cascade (D-12): If a parent (room or group) is hidden, all children render their eye in the **parent-hidden cascade state** (`text-text-ghost @ opacity-50`, `EyeOff` glyph). Children's own `hiddenIds` membership is **preserved** but visually overridden â€” toggling the parent back on restores children to their previously-toggled state.
- Eye-icon click does NOT change selection (event must call `stopPropagation`).
- 3D viewport ONLY (D-11): 2D Fabric canvas ignores `hiddenIds` â€” every wall/product/custom element/ceiling continues to render in 2D.
- Transient (D-13): `hiddenIds` resets to empty `Set` on page reload. NOT persisted to localStorage, NOT included in autosave snapshot, NOT pushed to undo history.

### Expand / collapse

- Per-room expanded state persists to localStorage keyed `gsd:tree:room:{roomId}:expanded` (mirrors Phase 33 `CollapsibleSection` behavior â€” D-03).
- Default (no localStorage entry): active room expanded, others collapsed (D-02).
- Group headers (Walls/Ceiling/Products/Custom elements) â€” **always expanded by default, no persistence**. Rationale: only 4 groups; collapsing them adds friction without payoff. (Planner discretion: if testing reveals taller rooms need group collapse, ship as a follow-up â€” not in TREE-01 scope.)

### Motion (D-39)

| Action | Default motion | Reduced-motion variant |
|--------|---------------|------------------------|
| Per-room expand/collapse | 150ms `ease-out` height transition (children fade-in over the same window via `opacity 0â†’100`) | Snap â€” no transition; children appear/disappear instantly |
| Camera focus dispatch (downstream of click) | Phase 35 easeInOutCubic 600ms tween | Snap (Phase 35 already handles this â€” tree dispatches the same shape, no new motion code) |
| Hover bg-color change | 150ms `transition-colors` | Snap (`useReducedMotion` returns `true` â†’ omit `transition-colors`) |
| Eye-icon glyph swap | None â€” instant glyph swap, no transform | Same (no motion to suppress) |

**Implementation:** wrap motion-bearing classNames in a `useReducedMotion()` check at the component top. When `matches === true`, omit `transition-*` and `duration-*` utilities and render expanded children unconditionally without the height-animation wrapper.

### Keyboard

- `Tab` traverses interactive elements in DOM order (chevron â†’ label â†’ eye-icon â†’ next row).
- `Space` / `Enter` on the row activates label click (select + focus).
- `Space` / `Enter` on the eye-icon activates visibility toggle.
- `Space` / `Enter` on the chevron activates expand/collapse.
- No custom arrow-key tree navigation in v1 (out of scope; revisit if a11y feedback requests it).

### Drag affordance

**NONE.** Drag-to-reparent is explicitly out of scope for TREE-01. Cursor is `cursor-pointer` on rows (not `cursor-grab` â€” that would mislead).
