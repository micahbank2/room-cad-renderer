<objective>
Fix the Phase 32 wallpaper / wall-art / custom-upload floor-texture regression: after applying a texture in 3D then toggling to 2D and back to 3D, the texture disappears. Root cause is Plan 32-03's D-05 cache migration — the refcount cache disposes textures synchronously when refs hit 0, which happens whenever ThreeViewport unmounts on a view-mode toggle. On remount, re-acquire kicks off a fresh TextureLoader fetch; during the load gap the overlay renders no texture, and in some paths the newly-loaded texture never re-drives the consuming material because the React state update raced with remount lifecycle. Pre-Phase-32 behavior was "wallpaper/wallArt caches never disposed" (module-level Maps held strong refs forever).

Purpose: Restore the v1.6 user-facing guarantee that view toggles do not drop applied textures, while preserving D-05's "single shared cache" intent and D-16's "refs=0 → dispose eventually" GPU-memory contract.

Output:
- Debounced-disposal `releaseTexture` — refs=0 schedules dispose after a grace window (default 3000 ms) rather than disposing synchronously; re-acquire within the grace window cancels the pending dispose and reuses the cached texture.
- Regression test covering "unmount+remount within grace reuses same Texture".
- Human-verify checkpoint where Jessica confirms the three failure modes (wallpaper, wallArt, custom-upload floor) survive the view toggle AND that the PBR paths still render.
</objective>

         ```typescript
         if (entry.refs <= 0) {
           // Don't dispose synchronously — view-mode toggles unmount+remount within a frame.
           // Schedule dispose; acquireTexture() will cancel if re-acquired within grace.
           if (pendingDispose.has(url)) {
             clearTimeout(pendingDispose.get(url)!);
           }
           const timer = setTimeout(() => {
             pendingDispose.delete(url);
             const current = cache.get(url);
             // Guard: refs may have gone back up via acquire during the grace window;
             // only dispose if still at 0.
             if (current && current.refs <= 0) {
               current.tex.dispose();
               cache.delete(url);
             }
           }, disposeGraceMs);
           pendingDispose.set(url, timer);
         }
         ```
