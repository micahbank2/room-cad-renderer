<objective>
Extend the Material data pipeline so AO and displacement maps can flow from upload to render. This is the data foundation — no UI, no 3D binding yet.

Purpose: The pipeline already exists for roughness and reflection maps (Phase 67/68). PBR-01 and PBR-02 require the exact same plumbing layered onto two new map slots. Without this foundation, no upload UI or mesh consumer can carry the new ids end-to-end.

Output: Material type + materialStore + surfaceMaterial resolver + useResolvedMaterial hook all carry aoMapId and displacementMapId through the same pattern as roughness/reflection. IDB is schemaless, so no migration is needed — existing Materials simply have these fields undefined.
</objective>
