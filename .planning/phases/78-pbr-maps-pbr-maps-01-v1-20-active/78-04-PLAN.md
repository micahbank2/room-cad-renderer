Design system constraints (CLAUDE.md §Design System):
- Pascal-aligned semantic tokens only: `bg-accent/20`, `text-muted-foreground`, `bg-card`, etc. — no `bg-obsidian-*`, no `text-text-*`
- Lucide-react icons only (D-A5) — no icons needed for the badges (text-only is the right call here for compactness)
- Use `font-mono` for the badge labels (D-10: short uppercase data identifiers like COLOR/ROUGH/AO)
- Use `rounded-smooth` or `rounded` for the badge corner shape
- Sentence-case body chrome — but these badges are short data tokens (COLOR, ROUGH, REFL, AO, DISP) so UPPERCASE is correct per D-10

RESEARCH.md §Pattern 5 anti-patterns to avoid:
- Do NOT fetch textures to check presence — `material.aoMapId` truthy/falsy is enough
- Do NOT show the COLOR badge for paint Materials (`material.colorHex` is set, `material.colorMapId` is unset)
- Do NOT show all 5 badges always — render conditionally

Pattern #7 (StrictMode-safety): No module-level registry writes in this plan. The presence checks are pure render-time computations. The existing `useEffect` in MaterialCard (blob-URL fetch for the thumbnail) already has correct cleanup with identity check; this plan does not touch it.
