    Read the full Toolbar.WallCutoutsDropdown.tsx file first.

    Add a `direction?: "up" | "down"` prop (default `"down"` for backward compat with any existing callers).

    In the positioning logic that computes `setPos(...)`, branch on direction:
    - direction === "down" (default): `{ top: rect.bottom + 4, left: rect.left }` (existing behavior, unchanged)
    - direction === "up": `{ top: rect.top - 4, left: rect.left }` with a `transform: translateY(-100%)` applied to the dropdown container's inline style

    If the dropdown container already has a `style` prop for positioning, add `transform: direction === "up" ? "translateY(-100%)" : undefined` to that style object.

    This is the fix for Pitfall 1 (RESEARCH.md): when the trigger button is at the bottom of the screen, `rect.bottom + 4` would place the dropdown off-screen. The upward direction uses `rect.top - 4` + translateY(-100%) to anchor to the top of the button and grow upward.

    Do NOT change any other behavior.
  </action>
