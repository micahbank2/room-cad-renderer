  <what-built>
    Phase 29 feet+inches editable dim-labels:
    - Dblclicking a wall's dimension label opens an inline input at the label
    - Input is 96px wide, pre-filled with feet+inches notation (e.g. `12'-6"`), select-all on focus
    - `validateInput` accepts `12'6"`, `12' 6"`, `12'-6"`, `12'`, `6"`, `12ft 6in`, `12 ft 6 in`, `12.5`, `12`, `12.5ft`; rejects `12 6`, `-5`, `0`, `0'0"`, `abc`
    - Commit on Enter OR blur; Escape cancels; invalid input silently cancels
    - PropertiesPanel LENGTH row accepts the same grammar
    - Each edit = exactly one undo entry
  </what-built>
  <how-to-verify>
    1. `npm run dev` and open the app in a browser.
    2. Draw four walls at roughly 0°, 45°, 90°, 135°.
    3. For EACH wall: double-click its dimension label. Confirm the input overlay appears visually CENTERED over the label text (not offset), regardless of wall angle.
    4. On one of the walls, type `12'6"` and press Enter. Confirm the wall resizes (pivoting from start) and the new label reads `12'-6"`.
    5. Dblclick the same label again, type a new value but click elsewhere on the canvas instead of pressing Enter. Confirm the wall resizes (commit-on-blur).
    6. Dblclick again, type `12 6` (no units), click away. Confirm the wall did NOT change (silent reject).
    7. Dblclick a wall label, note the current length. Type a new length, press Enter. Confirm the wall resized.
    8. Press Ctrl+Z ONCE. Confirm the wall returns to its original length in a single keystroke (no partial revert).
    9. Dblclick any label, press Escape immediately. Confirm no change and no undo entry (Ctrl+Z reverts an earlier edit, not the escape).
  </how-to-verify>
  <resume-signal>Type "approved" to continue, or describe any issue (overlay offset, blur feels wrong, multi-step undo, visual glitch)</resume-signal>
