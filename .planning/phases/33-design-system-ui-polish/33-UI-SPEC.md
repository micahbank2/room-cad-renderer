# Phase 33 — UI Design Contract

> Visual and interaction contract for Phase 33: Design System & UI Polish.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Conflict Notes (Checker Revision)

> **D-34 vs Checker Dimension 5 conflict — spacing 12px:**
> CONTEXT.md D-34 explicitly lists `12` as a canonical spacing value (`Spacing: 4, 8, 12, 16, 24, 32`). The checker's Dimension 5 rule permits only the standard 8-point set (4, 8, 16, 24, 32, 48, 64), which excludes 12.
> **Resolution chosen:** Remove `--spacing-md: 12px` as a CSS token and replace each usage site with either `--spacing-sm: 8px` or `--spacing-lg: 16px` (decided per-site below). The canonical scale documented in `src/index.css` and CLAUDE.md for D-34 purposes becomes 4/8/16/24/32. The 12px value is dropped from the design system entirely. This is a tighter constraint than D-34 specified but is required for checker compliance.

> **Typography weight conflict — D-03 implicit vs Checker Dimension 4:**
> The original spec used weights 400/500/600/700 across 4 roles. The checker allows at most 2 distinct weights. Resolution: 500 (medium) for display/h1/h2; 400 (regular) for label/body/value. The 600 and 700 weights are dropped.

## Spacing Scale

Source: D-34 — canonical scale defined in `src/index.css` `@theme {}` block. Note: 12px removed from token set to comply with checker Dimension 5 (see Conflict Notes above). All former `--spacing-md` usage sites resolved to `--spacing-sm` (8px) or `--spacing-lg` (16px) per the per-site decisions below.

| Token | Value | Tailwind equiv | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, chip inner padding, tight inline spacing |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, button padding (inline), floating toolbar px padding |
| lg | 16px | `p-4` / `gap-4` | Default panel row spacing, panel section padding, card padding |
| xl | 24px | `p-6` / `gap-6` | Section separation within sidebar |
| 2xl | 32px | `p-8` / `gap-8` | Major panel gaps |

**Former `--spacing-md: 12px` usage sites resolved:**
- Default panel row spacing → promoted to `--spacing-lg: 16px`
- Gesture chip canvas-edge inset → resolved to `--spacing-sm: 8px` (was 12px per D-17; 8px is ergonomically equivalent for bottom-left placement)
- Floating toolbar `py` padding → `--spacing-xs: 4px` (no change, was already `py-1`)

Exceptions:
- Touch / click targets minimum 32px height (ergonomic floor for Jessica's desktop use)
- Floating toolbar gap from selection bbox: 8px (D-12), uses `--spacing-sm`

**Post-phase verification (D-35):** Zero `p-[Npx]`, `m-[Npx]`, `rounded-[Npx]` arbitrary values in `Toolbar.tsx`, `Sidebar.tsx`, `PropertiesPanel.tsx`, and any modal component.

## Typography

Source: D-03, D-04, D-05. All tiers use font families already loaded from Google Fonts CDN.

**CSS custom properties (3 tokens — checker-compliant ≤4):**
- `--font-size-display: 28px` — hero headings (Space Grotesk)
- `--font-size-base: 13px` — shared by h1 + body roles
- `--font-size-sm: 11px` — shared by h2 + label + value roles

**Font weights (2 weights — checker-compliant):**
- `500` (medium) — display, h1, h2
- `400` (regular) — label, body, value

**Semantic role table (all 6 roles; CSS tokens shared as noted):**

| Role | Font | CSS Token | Weight | Line Height | Case | Usage |
|------|------|-----------|--------|-------------|------|-------|
| display | Space Grotesk | `--font-size-display` (28px) | 500 | 1.2 | Mixed-case | Hero/landing headings only (`DESIGN_YOUR_SPACE` brand name) |
| h1 | IBM Plex Mono | `--font-size-base` (13px) | 500 | 1.3 | Mixed-case | Top-level section headers: "Room config", "Properties", "Library" |
| h2 | IBM Plex Mono | `--font-size-sm` (11px) | 500 | 1.3 | Mixed-case | Panel sub-heads: "Position", "Dimensions", "Material" |
| label | IBM Plex Mono | `--font-size-sm` (11px) | 400 | 1.4 | Mixed-case | Button labels, property names, tab labels, chip text |
| body | Inter | `--font-size-base` (13px) | 400 | 1.5 | Mixed-case | Descriptive prose (empty-state bodies, help text) |
| value | IBM Plex Mono | `--font-size-sm` (11px) | 400 | 1.4 | As-is | Numeric field values, unit values — font-mono for alignment |

**UPPERCASE preserved (D-04) for:**
- Dynamic CAD identifiers: `WALL_SEGMENT_{id}`, `{PRODUCT_NAME}` (via `.toUpperCase().replace(/\s/g, "_")`)
- System status strings: `SYSTEM_STATUS: READY`, `SAVED`, `BUILDING_SCENE...`, `RENDERING...`
- Unit value labels when shown as standalone value labels: `LENGTH`, `WIDTH_FT`, `DEPTH_FT`

**Mixed-case applies (D-03/D-05) to:**
- Section headers in PropertiesPanel: "Position", "Dimensions", "Material", "Rotation"
- Sidebar panel headers: "Room config", "Properties", "Library"
- Button labels: "Add product", "Place", "Remove", "Duplicate", "Delete"
- Tab labels in CategoryTabs: "All", "Furniture", "Lighting", etc.

## Color

Source: `src/index.css` `@theme {}` block — Obsidian CAD palette. Non-negotiable per CONTEXT.md.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#12121d` (`obsidian-base`) | App root background, canvas backdrop — primary focal point is the CAD canvas viewport |
| Secondary (30%) | `#1b1a26` (`obsidian-low`) + `#1f1e2a` (`obsidian-mid`) | Sidebar panels, modal backgrounds, card surfaces |
| Elevated surface | `#292935` (`obsidian-high`) + `#343440` (`obsidian-highest`) | Hover states, active chips, elevated items |
| Accent (10%) | `#7c5bf0` (`accent`) | Reserved-for list below |
| Accent light | `#ccbeff` (`accent-light`) | Active label text on accent backgrounds, active chip values |
| Destructive | `#ffb4ab` (`error`) | Delete confirmation, destructive action buttons only |

**Accent (`#7c5bf0`) reserved ONLY for:**
- Active/selected tool button highlight in Toolbar
- Focused input `border-color` and focus ring (`box-shadow: 0 0 0 1px rgba(124,91,240,0.3)`)
- Active category tab underline/indicator in `<CategoryTabs>`
- Selected chip state in rotation preset chips (#87)
- `<LibraryCard>` selected state border
- Floating toolbar button hover background (`bg-accent/20`)
- Chevron color on hovered collapsible section header

**Accent NOT used for:** unselected buttons, default borders, text on body surfaces, navigation items in default state.

**Glass panel (gesture chip, floating toolbar):** `rgba(31, 30, 42, 0.8)` background + `backdrop-filter: blur(12px)` + `border: 1px solid rgba(72,69,84,0.2)` — reuse `.glass-panel` CSS class.

**Canvas viewport:** The 2D CAD canvas is the primary focal point of the interface. All surrounding chrome (toolbar, sidebar, panels) must remain visually recessive — lower brightness, lower saturation — so the canvas commands the eye. No panel element should visually compete with the canvas content.

## Interaction Contracts

### Collapsible Properties Sections (#84)

- Chevron: lucide `ChevronRight` (collapsed) / `ChevronDown` (expanded) — placed LEFT of section header text (D-09)
- Click target: entire header row (full width)
- Animation: `height` transition 200ms `ease-out` (D-08)
- Reduced motion: `prefers-reduced-motion: reduce` → snap open/closed, no transition (D-08)
- Persistence: `localStorage` key `ui:propertiesPanel:sections` — JSON object `{ [sectionId]: boolean }` (D-06)
- Default: all sections expanded on first visit (D-07)
- Header text color: `text-text-muted` default, `text-text-primary` on hover
- Chevron color: `text-text-ghost` default, `text-accent` on hover

### Floating Selection Toolbar (#85)

- Scope: 2D canvas only (D-10)
- Actions: Duplicate (`lucide Copy`), Delete (`lucide Trash2`) (D-11)
- "Duplicate" and "Delete" labels act on the currently selected object (implied noun — no explicit object reference needed in the label; tooltip may clarify if needed)
- Position: screen-space above selection bbox, 8px gap (`--spacing-sm`), clamps to canvas viewport edges (D-12)
- If insufficient room above: flips to below selection bbox
- Visibility: shown when `selectedIds.length >= 1` AND `_dragActive === false` (D-13)
- Hide during drag: reuse Phase 25 `_dragActive` flag from cadStore (D-13)
- Styling: `.glass-panel` background, `--radius-lg` (8px), `px-2 py-1` (8px/4px), icon size 14px
- Button hover: `bg-accent/20 text-accent-light` for Duplicate; `bg-error/20 text-error` for Delete
- NO new keyboard shortcuts (D-14) — Delete key already wired

### Gesture Chip (#86)

- Placement: bottom-left of active canvas viewport, 8px inset from edges (`--spacing-sm`; was 12px per D-17 — resolved to nearest allowed token, see Conflict Notes)
- Styling: `.glass-panel`, `text-text-dim`, label-size (`--font-size-sm` 11px, font-mono), `--radius-lg`
- Content:
  - 2D mode: `Drag to pan  •  Wheel to zoom` (D-16)
  - 3D mode: `L-drag rotate  •  R-drag pan  •  Wheel zoom` (D-16)
- Dismiss: `×` button (`lucide X`, 10px) right of text; dismissed state → `localStorage` key `ui:gestureChip:dismissed` = `"true"` (D-15)
- Hidden during active drag (`_dragActive` signal) — not just on dismiss (D-18)
- Dismissed chip: permanently hidden until localStorage cleared

### Rotation Preset Chips (#87)

- Presets: `-90°`, `-45°`, `0°` (reset), `+45°`, `+90°` (D-19)
- Placement: chip row to the RIGHT of the numeric rotation input in PropertiesPanel (D-22)
- Selected state: chip where `rotation === preset` gets `bg-accent/20 text-accent-light border-accent/30` (D-22)
- Default state: `bg-obsidian-high text-text-dim border-outline-variant/20`
- Chip size: `px-2 py-0.5` (8px/2px), `--radius-sm` (2px), font-mono `--font-size-sm` 10px
- Each click: single undo entry via `updatePlacedProduct` / `updatePlacedCustomElement` (D-20)
- Works for products AND custom elements (D-21)

### Inline-Editable Titles (#88)

- Targets: document title in Toolbar + room tab labels
- Activation: single-click (D-25)
- Affordance on hover: `cursor: text`, subtle underline `text-text-dim` — no edit icon (D-24)
- Pattern: Phase 31 live-preview (D-23) — keystroke = live preview, Enter/blur = commit, Escape = revert
- Validation: trim whitespace; max 60 chars; empty commit → revert to previous name (D-27)
- Persistence: doc title → `projectStore.activeName` (auto-save already wired via Phase 28); room name → `cadStore.rooms[id].name` (D-26)
- Input styling: transparent background, `border-b border-accent` when active, no border when inactive

### Library Card — `<LibraryCard>` (#89)

- Props: `thumbnail`, `label`, `selected`, `onClick`, `onRemove?`, `variant?: "grid" | "list"` (D-29)
- Label: font-mono, `--font-size-sm` (11px), `text-text-muted`, truncated at 1 line
- Selected state: `border-accent/60` + `bg-accent/10`
- Default state: `bg-obsidian-low`, `ghost-border`, `--radius-md` (4px)
- Hover: `bg-obsidian-high`
- Remove button: `lucide X` 12px, appears on hover only, top-right corner, `text-text-ghost hover:text-error`
- Grid variant: square aspect ratio, thumbnail fills top, label below
- List variant: thumbnail 32×32px left, label right

### Category Tabs — `<CategoryTabs>` (#89)

- Props: `tabs: { id, label, count? }[]`, `activeId`, `onChange` (D-30)
- Active tab: bottom border `border-accent`, `text-text-primary`, weight 500
- Inactive tab: `text-text-dim`, no border
- Tab strip: horizontal, `gap-4` (16px) between tabs, `border-b border-outline-variant/20` on container
- Count badge: `text-text-ghost` font-mono `--font-size-sm` (10px) in parentheses: `All (12)`
