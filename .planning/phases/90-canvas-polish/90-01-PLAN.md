    Body:
    ```tsx
    // Phase 90 D-03 (#201): defer canvasTheme read until after <html class="dark">
    // has flushed to the DOM. useTheme's class-write effect and our redraw
    // effect may run in either order in the same commit; if redraw wins,
    // getCanvasTheme() reads stale CSS tokens and Fabric paints the wrong
    // backdrop. rAF guarantees we're past the class write before re-probing.
    // Direct fc mutation (NOT a full redraw) to avoid interrupting selectTool
    // drag transactions.
    useEffect(() => {
      const raf = requestAnimationFrame(() => {
        const fc = fcRef.current;
        if (!fc) return;
        const theme = getCanvasTheme();
        setFabricSyncTheme(theme);
        fc.backgroundColor = theme.background;
        fc.renderAll();
      });
      return () => cancelAnimationFrame(raf);
    }, [resolved]);
    ```

    Do NOT remove the existing `resolved` dep from the redraw `useCallback` at line 295 — keep the full-redraw path for cases where theme changes coincide with other state changes. The rAF effect is an additional patch path that handles theme-only flips.
