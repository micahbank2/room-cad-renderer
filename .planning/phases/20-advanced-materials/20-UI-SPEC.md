## Typography

3 sizes, 2 weights. All UI text uses `font-mono` (IBM Plex Mono). No serif or display fonts in this phase.

| Role | Size | Weight | Line Height | Font | Tailwind |
|------|------|--------|-------------|------|----------|
| Micro | 8px | 400 | 1.2 | IBM Plex Mono | `text-[8px]` |
| Label | 10px | 400 | 1.2 | IBM Plex Mono | `text-[10px]` |
| Value | 11px | 400 | 1.3 | IBM Plex Mono | `text-[11px]` |

Usage mapping:
- **Micro (8px):** Swatch name labels, override notes (`OVERRIDDEN_BY_MATERIAL`, `OVERRIDDEN_BY_PAINT`)
- **Label (10px):** Section headers (`SURFACE_MATERIAL`, `CEILING_PAINT`, `FLOOR_MATERIAL`), active mode indicator text, CLEAR_MATERIAL button, UPLOAD_IMAGE button
- **Value (11px):** Active material indicator (`MATERIAL: PLASTER`), hex values, scale/rotation values

Source: Consolidated from existing FloorMaterialPicker.tsx and PropertiesPanel.tsx patterns. Original 7px and 9px sizes merged up to 8px and 10px respectively for perceptible differentiation.

## Color

|------|-------|-------|
| Dominant (60%) | `#12121d` (obsidian-base) | App background behind panels |
| Secondary (30%) | `rgba(31,30,42,0.8)` (glass-panel) | PropertiesPanel surface, SurfaceMaterialPicker container |

Accent reserved for:
- Active/selected material swatch border ring (`ring-1 ring-accent/30` + `border-accent`)
- Active mode indicator text (`text-accent-light`)

Additional color usage specific to this phase:
- Material swatch backgrounds use the material's own `color` hex value from the catalog
- Dimmed/inactive mode section uses `text-text-ghost` opacity
- Section dividers use `border-outline-variant/20`

Source: Existing PropertiesPanel.tsx, SwatchPicker.tsx, FloorMaterialPicker.tsx patterns.

## Focal Points

| Component | Primary Focal Point | Rationale |
|-----------|-------------------|-----------|
| SurfaceMaterialPicker | The currently-selected swatch (accent border + ring glow) | User needs to instantly see which material is active among the grid |
| CeilingPaintSection (full layout) | The active section header in `text-accent-light` | The accent-colored header signals which mode (material vs. paint) is currently controlling the ceiling; the dimmed section recedes |
| FloorMaterialPicker (updated) | The active preset swatch (accent border) | Same pattern as SurfaceMaterialPicker -- selected state draws the eye |

When no material or paint is selected, no focal point activates -- both sections appear at equal visual weight, inviting the user to choose.

## Component Inventory

### New Components

#### SurfaceMaterialPicker

**Purpose:** Reusable swatch grid for selecting floor or ceiling surface materials from the unified catalog.

**Location:** `src/components/SurfaceMaterialPicker.tsx`

**Props:**
```typescript
interface Props {
  surface: "floor" | "ceiling";
  activeId: string | undefined;
  onSelect: (id: string | undefined) => void;
}
```

**Layout:** 4-column grid of square color swatches. Each swatch is a button containing:
1. A square color preview div (`aspect-square`, background-color from material catalog)
2. A mono label below the swatch (`text-[8px] text-text-dim`, truncated, UPPER_SNAKE_CASE)

**States:**
| State | Visual |
|-------|--------|
| Default | `border border-outline-variant/20 rounded-sm` |
| Selected/active | `border-accent ring-1 ring-accent/30` |
| Hover | `border-outline-variant/40` (subtle brightening) |

**Interaction:** Click toggles selection. Clicking the active swatch deselects it (passes `undefined` to `onSelect`).

**Grid dimensions:** 4 columns. Floor shows 9 swatches (8 presets + CONCRETE shared). Ceiling shows 4 swatches (PLASTER, WOOD_PLANK, PAINTED_DRYWALL, CONCRETE). Grid auto-rows, no fixed height.

**Swatch size:** Each swatch button gets `p-1` (4px) internal padding. The color square fills available width. Label is `mt-1` (4px) below the square.

### Modified Components

#### CeilingPaintSection

**Current:** Shows `CEILING_PAINT` header, SwatchPicker, and LIME_WASH checkbox.

**Change:** Add a `SurfaceMaterialPicker` section above the paint section. The two sections are mutually exclusive:

**Layout (top to bottom):**
1. **SURFACE_MATERIAL** header (`text-[10px] text-accent-light tracking-widest`)
2. SurfaceMaterialPicker with `surface="ceiling"`
3. Active material indicator: `MATERIAL: {LABEL}` in `text-[10px] text-accent-light` when a material is active, or nothing when no material is selected
4. CLEAR_MATERIAL button (`text-[10px] text-text-ghost hover:text-text-primary tracking-widest`) -- only visible when a material is active
5. Divider: `border-t border-outline-variant/20 pt-3 mt-3`
6. **CEILING_PAINT** header (existing)
7. SwatchPicker (existing) -- visually dimmed when surfaceMaterialId is active: wrapper gets `opacity-40 pointer-events-none`
8. LIME_WASH checkbox (existing) -- same dimming rule

**Mutual exclusion behavior:**
- When user selects a surface material: store action clears `paintId` and `limeWash`. Paint section dims.
- When user selects a paint: store action clears `surfaceMaterialId`. Material picker deselects.
- Dimmed section shows a small note: `text-[8px] text-text-ghost` reading `OVERRIDDEN_BY_MATERIAL` or `OVERRIDDEN_BY_PAINT` respectively.

#### FloorMaterialPicker

**Current:** Dropdown `<select>` with preset options + custom upload + scale/rotation controls.

**Change:** Replace the preset `<select>` dropdown with a `SurfaceMaterialPicker` grid (surface="floor"), keeping custom upload and scale/rotation controls below.

**Layout (top to bottom):**
1. **FLOOR_MATERIAL** header (existing, unchanged)
2. SurfaceMaterialPicker with `surface="floor"` and `activeId` derived from current `FloorMaterial.presetId`
3. **UPLOAD_IMAGE** button: `font-mono text-[10px] text-text-ghost tracking-widest border border-outline-variant/20 rounded-sm py-1 w-full` with hidden file input (existing logic, restyled from dropdown option to standalone button)
4. Active indicator: color swatch + hex value row (existing pattern, shown when preset or custom is active)
5. Scale + rotation controls (existing, unchanged)
6. RESET_TO_DEFAULT button (existing, unchanged)

#### PropertiesPanel (ceiling section)

**Current:** Shows ceiling height, vertices, then CeilingPaintSection.

**Change:** CeilingPaintSection now internally contains the material picker. No PropertiesPanel layout changes needed. The ceiling section simply renders `<CeilingPaintSection>` which now includes both material and paint.

## Interaction Contracts

### Material Selection (SurfaceMaterialPicker)

| Action | Result |
|--------|--------|
| Click unselected swatch | Swatch becomes active (accent border). Calls `onSelect(materialId)`. |
| Click already-active swatch | Swatch deselects. Calls `onSelect(undefined)`. |
| Click swatch in ceiling picker | `setCeilingSurfaceMaterial` action fires, clears paintId/limeWash. Paint section dims. |
| Click swatch in floor picker | `setFloorMaterial` action fires with `kind: "preset"`, matching existing behavior. |

### Mutual Exclusion (Ceiling only)

| Action | Material Picker State | Paint Picker State |
|--------|----------------------|-------------------|
| Select material | Active swatch highlighted | `opacity-40 pointer-events-none`, note: `OVERRIDDEN_BY_MATERIAL` |
| Select paint | No swatch highlighted | Active, paint swatch highlighted |
| Clear material | No swatch highlighted | Enabled (full opacity) |
| Clear paint | Enabled (if was overridden) | No swatch highlighted |
| Neither active | Enabled | Enabled |

### Floor Material Picker (updated)

| Action | Result |
|--------|--------|
| Click preset swatch | Floor material set to preset. Upload state cleared if was custom. |
| Click UPLOAD_IMAGE button | File picker opens. On image select, floor material becomes custom. Preset swatch deselects. |
| Click RESET_TO_DEFAULT | Floor material cleared. All swatches deselect. Scale/rotation controls hide. |
