<objective>
Phase 67 establishes the Material entity as a first-class data system in Room CAD Renderer. Jessica can upload a Material (color/roughness/reflection texture maps + name, brand, SKU, cost, lead time, tile size) and it persists in her library across reloads, dedupes on identical color-map upload, and shows metadata on hover. The Material lives in its own IndexedDB store (`room-cad-materials`), wraps existing UserTexture references (D-09 RESOLVED to wrapper architecture per RESEARCH.md), and has zero impact on cadStore / project snapshots.

Purpose: This is the foundation for Phases 68 (Material Application — apply Materials to surfaces), 69 (Product–Material Linking), and 70 (Library Restructure). The wrapper architecture is load-bearing — Phase 68's surface renderers already consume `userTextureId` references, so Material applies for free if Materials store color-map IDs rather than blobs.

Output: 6 new source files, 1 modified file (`ProductLibrary.tsx`), 5 new vitest files, 1 new Playwright e2e file. Single plan with task-level breakdown — Task 0 scaffolds tests (Wave 0 per VALIDATION.md), Task 1 lands data layer (types + store + hook), Task 2 lands UI layer (modal + cards + library section + ProductLibrary integration).
</objective>
