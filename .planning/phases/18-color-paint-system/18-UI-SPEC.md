## Visual Hierarchy

Primary focal point: the F&B catalog swatch grid. It is the densest, most colorful element in the panel and carries the primary browsing action. All other elements (hue filter chips, search input, recently-used row, MY_COLORS, lime wash toggle, apply button) are subordinate supporting controls arranged above and below it.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#12121d` (`obsidian-base`) | App root background, 2D canvas backdrop |
| Secondary (30%) | `#1b1a26` (`obsidian-low`) | Sidebar panel background, PAINT section container |
| Accent (10%) | `#7c5bf0` (`accent`) | Active filter chip border, active state indicators, section header text |
| Accent light | `#ccbeff` (`accent-light`) | Section header text `PAINT`, active swatch ring, applied paint indicator text |
| Destructive | `#ffb4ab` (`error`) | DELETE option in custom swatch right-click menu only |

Accent reserved for:
1. Active hue filter chip border (`border-accent`)
2. Active SIDE_A / SIDE_B toggle (existing pattern, `border-accent text-accent-light bg-accent/10`)
3. Currently applied paint swatch ring (`ring-1 ring-accent`)
4. Section header label `PAINT` (matches existing `text-accent-light` section header pattern)
5. APPLY_TO_ALL_WALLS button on hover (`hover:text-accent-light`)

All other interactive elements: `text-text-dim` at rest, `hover:text-accent-light` on hover. No accent on neutral resting states.

Paint swatches render in their actual F&B hex color — they are data, not UI chrome. Accent is not applied to swatches themselves.

Source: CONTEXT.md decisions + `WallSurfacePanel.tsx` + `src/index.css` token definitions.

## Interaction Contracts

### Swatch Application
- Click any swatch (F&B catalog or MY_COLORS) → applies immediately to selected wall side or ceiling
- No "confirm" step; matches existing wallpaper color picker behavior
- Source: CONTEXT.md D-02, D-05

### Swatch Hover Tooltip
- Hover a swatch for 300ms → small tooltip appears above the swatch
- Tooltip content: F&B name in `font-mono text-[8px] text-text-primary`
- Tooltip background: `bg-obsidian-highest border border-outline-variant/30 rounded-sm px-1 py-1`
- Custom colors: show user-assigned name
- Source: CONTEXT.md D-02

### Hue Filter Chips
- Click a chip → filters swatch grid to that hue family
- Click the same active chip again → clears filter, shows all 132 swatches
- Only one filter active at a time
- Active chip: `border-accent text-accent-light bg-accent/10`
- Inactive chip: `border-outline-variant/30 text-text-ghost`
- Source: CONTEXT.md D-03

### Name Search
- Typing in SEARCH_COLORS input filters the F&B grid in real time (case-insensitive contains match)
- Search and hue filter compose: both active simultaneously narrows results further
- Empty result: show `NO_COLORS_FOUND` in `font-mono text-[9px] text-text-ghost text-center py-2`

### Recently Used Row
- Always visible at top of PAINT section regardless of filter/search state
- Shows last 8 applied colors, most recent at left
- Clicking a recently-used swatch applies it to the current wall side — same as any catalog swatch
- Persists in cadStore snapshot (undo/redo aware)
- Source: CONTEXT.md D-04, R-01

### Add Custom Color (inline expand)
- "+ ADD_COLOR" button expands an inline row directly below MY_COLORS swatches
- Row contains: name text input + react-colorful HSL wheel (compact, ~160px wide) + SAVE_COLOR button
- SAVE_COLOR is disabled until name is non-empty; shows `text-text-ghost cursor-not-allowed` when disabled
- SAVE_COLOR enabled: `text-accent-light hover:text-accent`
- Pressing Enter in name field saves if color is set
- Source: CONTEXT.md D-11

### Delete Custom Color
- Right-click (desktop) or long-press 600ms (touch) on a custom swatch
- Shows a micro-menu: one item `DELETE` in `text-error text-[9px] font-mono`
- No confirmation dialog — matches CONTEXT.md D-12 decision
- Micro-menu: `bg-obsidian-highest border border-outline-variant/30 rounded-sm px-2 py-1`
- Source: CONTEXT.md D-12

### Lime Wash Toggle
- Checkbox per-placement: checked = lime wash finish on this wall side / ceiling
- In 2D: renders a stippled/cloudy CSS pattern overlay at 20% opacity over the wall color fill
- In 3D: `roughness: 0.95` on `meshStandardMaterial`
- Toggle state is independent of which catalog color is applied
- Source: CONTEXT.md D-13, D-14, D-15, R-04

### Apply to All Walls
- Button at bottom of paint section
- One click, no confirmation dialog
- Paints every wall's currently-active side (SIDE_A or SIDE_B per the active toggle) in the current room
- Feedback: recently-used row updates to reflect the applied color
- Source: CONTEXT.md D-07, Claude's Discretion note on scope

### 2D Canvas Lime Wash Pattern
- Implementation via CSS/canvas repeating pattern: small random-offset dots or crosshatch at 20% opacity
- Pattern color: white (`rgba(255,255,255,0.20)`) overlaid on the wall's paint hex fill
- Rendered in `fabricSync.ts` as a Fabric overlay object on top of the colored wall polygon
- Source: CONTEXT.md D-13, Claude's Discretion

## Swatch Grid Dimensions

| Property | Value | Rationale |
|----------|-------|-----------|
| Swatch size | 20×20px | Fits 8 per row in ~200px sidebar width with 4px gaps |
| Grid gap | 4px (`gap-1`) | Multiple of 4; dense paint-chip feel; `8 × 20 + 7 × 4 = 188px` — fits within `p-3` sidebar |
| Swatches per row | 8 | `(8 × 20) + (7 × 4) = 188px` — fits within `p-3` sidebar |
| Hue filter chip size | 14×14px | Compact row of 7 chips + labels |
| Recently-used swatch | 18×18px | Slightly smaller to distinguish from catalog swatches |
| Catalog max height | 160px (`max-h-40`) | Scrollable; shows ~5 rows before scroll |
| MY_COLORS max height | 40px | 1 row; expands with custom color count |

## States Required

Each interactive element must handle these states:

### Paint Swatch
| State | Visual |
|-------|--------|
| Default | Solid color fill, no border |
| Hover | `ring-1 ring-white/30` + tooltip appears after 300ms |
| Applied (current) | `ring-2 ring-accent ring-offset-1 ring-offset-obsidian-low` |
| Filtered out | Not rendered (removed from DOM, not hidden) |

### Hue Filter Chip
| State | Visual |
|-------|--------|
| Default | `border-outline-variant/30 text-text-ghost bg-transparent` |
| Hover | `border-outline-variant/60 text-text-dim` |
| Active | `border-accent text-accent-light bg-accent/10` |

### APPLY_TO_ALL_WALLS Button
| State | Visual |
|-------|--------|
| Default | `border border-outline-variant/30 text-text-dim py-2` |
| Hover | `border-accent/50 text-accent-light` |
| No paint applied | `opacity-40 cursor-not-allowed` (nothing to apply) |

### SAVE_COLOR Button (custom color)
| State | Visual |
|-------|--------|
| Disabled (no name) | `text-text-ghost cursor-not-allowed` |
| Enabled | `text-accent-light hover:text-accent cursor-pointer` |

### Lime Wash Checkbox
| State | Visual |
|-------|--------|
| Unchecked | Standard dark checkbox, `accent-accent` |
| Checked | Checked; 2D canvas updates to show stipple overlay |

## Implementation Notes for Executor

1. Insert PAINT section into `WallSurfacePanel.tsx` below the WALLPAPER section, above WAINSCOTING. Use the same `<div>` + sub-label + control row pattern.

2. Ceiling paint is a separate section — either in a new `CeilingPaintPanel.tsx` that appears in PropertiesPanel when a ceiling is selected, or as a new sub-section in an existing ceiling properties component.

3. The swatch grid container must be `overflow-y-auto max-h-40` with a custom scrollbar styled to match: `scrollbar-thin scrollbar-thumb-obsidian-highest scrollbar-track-transparent`.

4. react-colorful picker renders inside the inline expand row. Constrain its width: `w-40` (160px) to fit the sidebar. Use `HexColorPicker` (not HSL) — hex is what Jessica will paste from Pinterest.

5. The lime wash 2D pattern: use a `fabric.Pattern` with a small canvas tile showing random white dots at 20% opacity. Apply as `fill` pattern on the wall polygon, composited over the paint color using a separate overlay polygon at `opacity: 0.2`.

6. Tooltip implementation: use a `title` attribute as a baseline for accessibility, plus a custom positioned div toggled via `onMouseEnter`/`onMouseLeave` with a 300ms delay (`setTimeout`).

7. All labels follow `SCREAMING_SNAKE_CASE` — no sentence case or title case anywhere in this panel.

8. `font-mono text-[9px] tracking-widest uppercase` is the standard label class in this codebase — use it verbatim.
