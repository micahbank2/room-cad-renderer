# Phase 5.1 Plan 00: v1.0 Integration Gaps Summary

**One-liner:** Closed three v1.0 audit gaps — startup project hydration from IndexedDB, walk-camera respawn on active-room change, and orbit-camera position restore on mode toggle.

## Changes

### Task 1 — Startup Project Hydration (SAVE-02)
**Commit:** `6aca2d4`
**Files:** `src/App.tsx`

Added a mount-time `useEffect` that calls `listProjects()` (returns projects sorted by `updatedAt` DESC), picks the most recent, and hydrates both stores via `useCADStore.getState().loadSnapshot(latest.snapshot)` and `useProjectStore.getState().setActive(latest.id, latest.name)`. Fresh installs with no saved projects keep the default "Main Room" state. Uses a `cancelled` flag to prevent race if component unmounts mid-load. `loadSnapshot` already runs `migrateSnapshot` internally (Phase 5 D-15) so v1 legacy snapshots auto-upgrade.

### Task 2 — Walk Camera Respawn on Room Switch (VIZ-05)
**Commit:** `f7781b0`
**Files:** `src/three/WalkCameraController.tsx`

Removed the `didSpawn.current` one-shot gate and replaced it with a `useEffect` keyed on `activeRoomId` (subscribed via `useCADStore((s) => s.activeRoomId)`). The effect re-centers camera at `{ width/2, 5.5, length/2 }` with yaw reset on mount AND on every subsequent room change. No other walk state (pointer lock, keys) is touched.

### Task 3 — Orbit Camera Position Restore (VIZ-05)
**Commit:** `502cb74`
**Files:** `src/three/ThreeViewport.tsx`

Initialized `orbitPosRef` to a non-null default `[halfW + 15, 12, halfL + 15]` (matching the `<Canvas camera={...}>` default). Added a `useEffect` keyed on `cameraMode` inside Scene that, when mode becomes "orbit", reads `orbitPosRef.current`, calls `ctrl.object.position.set(x, y, z)` on the OrbitControls ref, then `ctrl.update()` to flush the change. Target restoration was already working via `orbitTargetRef` → `<OrbitControls target={...} />` prop; position now has a symmetric path.
