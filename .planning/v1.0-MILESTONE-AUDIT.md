## Critical Integration Gaps

### 1. SAVE-02 reload/hydration missing (affects usability of entire v1)

`useAutoSave()` writes `{ version: 2, rooms, activeRoomId }` to IndexedDB on every change. But:
- `App.tsx` never calls `listProjects` or `loadProject` on mount
- `projectStore.activeId` is not persisted
- `cadStore` always starts with the hardcoded "Main Room" default on reload

**User impact:** All saved work is stranded in IDB after refresh. A new project is silently created alongside old data on every reload.

**Fix options:**
- A. Mount `ProjectManager` component in Sidebar (it's defined, just orphaned)
- B. Add `useEffect` to App.tsx: `listProjects()` → pick most recent → `loadProject(id)` → `useCADStore.getState().loadSnapshot(snapshot)` + `useProjectStore.getState().setActive(id)`

### 2. Walk mode across room switches (VIZ-05 × ROOM-01)

`WalkCameraController.didSpawn.current` blocks re-spawn when `activeRoomId` changes. Walking → switching rooms leaves camera stranded at old coords.

**Fix:** Replace `didSpawn.current` gate with `useEffect(() => { spawnCamera() }, [activeRoomId])`.

### 3. Orbit camera position restore incomplete (VIZ-05)

`orbitPosRef` is written via `onChange` handler but never read back to restore position when switching walk→orbit. Target IS restored, position is not.

**Fix:** Pass `orbitPosRef.current` into OrbitControls camera position on mount, or manually set `camera.position.set(...)` in a useEffect.
