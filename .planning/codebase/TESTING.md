## What Exists Instead of Tests

The codebase relies on:
- **TypeScript** (`typescript ^6.0.2`) for compile-time correctness across all files
- **Manual browser testing** — the dev server (`vite`) is the only runtime validation
- **Zustand store interfaces** as contracts — `CADState` and `UIState` in `src/stores/` define expected shape
- **Type imports** — `import type { ... }` enforces correct data shapes at boundaries

## Gaps by Layer

**Geometry utilities (`src/lib/geometry.ts`):**
- `snapTo`, `snapPoint`, `wallLength`, `formatFeet`, `constrainOrthogonal`, `wallCorners`, `closestPointOnWall` are pure functions with no side effects — highest-priority candidates for unit tests
- `uid()` uses a module-level counter — needs reset between tests

**Store logic (`src/stores/cadStore.ts`):**
- Undo/redo stack (`past`/`future`) has complex push/pop logic capped at `MAX_HISTORY = 50` — fragile without tests
- `removeSelected` deletes from both `walls` and `placedProducts` maps simultaneously — easy to regress
- `loadSnapshot` resets history arrays — no verification this happens correctly

**Canvas tools (`src/canvas/tools/`):**
- `wallTool.ts`, `doorTool.ts`, `windowTool.ts`, `productTool.ts`, `selectTool.ts` — interact with Fabric.js canvas events, making them hard to unit test without a DOM/canvas mock

**Serialization (`src/lib/serialization.ts`):**
- `saveProject`, `loadProject`, `deleteProject`, `listProjects` all use `idb-keyval` (IndexedDB) — would require `fake-indexeddb` or similar mock to test

**Components:**
- No component rendering tests
- Form validation logic in `AddProductModal.tsx` (submit disabled when `!name`) is untested

## Recommended Setup

### Vitest (preferred for this stack)

Install:
```bash
npm install -D vitest @vitest/ui jsdom @testing-library/react @testing-library/user-event
```

Configure in `vite.config.ts`:
```ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";
import path from "path";

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: { "@": path.resolve(__dirname, "./src") },
  },
  test: {
    environment: "jsdom",
    globals: true,
    setupFiles: ["./src/test/setup.ts"],
  },
});
```

Update `package.json` test script:
```json
"test": "vitest",
"test:ui": "vitest --ui",
"test:coverage": "vitest run --coverage"
```

### File Placement Convention (recommended)

Co-locate tests with source files:

src/

### Highest-Value Tests to Write First

**1. Pure geometry functions (`src/lib/geometry.ts`)** — no mocking needed:
```ts
import { describe, it, expect } from "vitest";
import { snapTo, formatFeet, constrainOrthogonal } from "@/lib/geometry";

describe("snapTo", () => {
  it("snaps to nearest 0.5 increment", () => {
    expect(snapTo(1.3, 0.5)).toBe(1.5);
    expect(snapTo(1.2, 0.5)).toBe(1.0);
  });
});

describe("formatFeet", () => {
  it("formats whole feet", () => expect(formatFeet(3)).toBe("3'"));
  it("formats feet and inches", () => expect(formatFeet(3.5)).toBe("3'-6\""));
  it("handles inch rollover", () => expect(formatFeet(3.99)).toBe("4'"));
});
```

**2. CAD store undo/redo (`src/stores/cadStore.ts`)** — use Zustand's store directly:
```ts
import { describe, it, expect, beforeEach } from "vitest";
import { useCADStore } from "@/stores/cadStore";

beforeEach(() => {
  useCADStore.setState({
    room: { width: 20, length: 16, wallHeight: 8 },
    walls: {}, placedProducts: {}, past: [], future: [],
  });
});

describe("undo/redo", () => {
  it("undoes wall addition", () => {
    useCADStore.getState().addWall({ x: 0, y: 0 }, { x: 10, y: 0 });
    expect(Object.keys(useCADStore.getState().walls)).toHaveLength(1);
    useCADStore.getState().undo();
    expect(Object.keys(useCADStore.getState().walls)).toHaveLength(0);
  });
});
```

**3. Serialization round-trip (`src/lib/serialization.ts`)** — mock `idb-keyval`:
```ts
import { vi } from "vitest";
vi.mock("idb-keyval", () => ({
  get: vi.fn(),
  set: vi.fn(),
  del: vi.fn(),
  keys: vi.fn(),
}));
```
