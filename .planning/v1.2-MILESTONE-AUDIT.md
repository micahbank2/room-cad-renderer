## Critical Gap: CUSTOM-05 Persistence

**Symptom:** Jessica creates a custom element → places it in a room → reloads the page → placed custom element disappears from 3D, orphan record remains in store.

**Root cause:** Two save call-sites explicitly omit the `customElements` top-level field from the snapshot passed to `saveProject`:

```ts
// useAutoSave.ts:36
await saveProject(id, name, { version: 2, rooms, activeRoomId });

// ProjectManager.tsx:33
await saveProject(id, projectName, { version: 2, rooms, activeRoomId });
```

The store's internal `snapshot()` helper at `cadStore.ts:81-91` DOES include customElements via `...(root.customElements ? { customElements: ... } : {})` — but save call-sites construct their own payload object and skip the spread.

**Fix (~5 lines):**
```ts
// Read customElements from cadStore.getState() and include in payload
const st = useCADStore.getState() as any;
await saveProject(id, name, {
  version: 2,
  rooms: st.rooms,
  activeRoomId: st.activeRoomId,
  customElements: st.customElements,
});
```

Plus extend autosave subscription to dirty on customElements changes.
